# Number

### Number Object

Number 메서드를 생성자 함수로써 호출할 경우, wrapper object를 생성한다.

인수로 입력된 값은 wrapper 객체의 [[NumberData]] 내부슬롯에 저장된다.

- 이전 ES5에서 [[NumberData]] 를 [[PrimitiveValue]] 라고 불렀다.

- [[PrimitiveValue]]는 원시값의 숫자 변수를 프로퍼티 접근 연산자(.)로 접근할 때 암묵적으로 생성되는 wrapper 객체와 같다.

```js
const num1 = new Number(10);
```

### IEEE 754 Double Precision

JS환경에서 Number 데이터는 IEEE 754 double precision 표현 방식을 사용해 표현된다.

IEEE 스펙에 의하면 총 64비트를 사용하여 하나의 값을 표현한다.

- signbit(1bit): 값의 기호를 결정

- exponent(11bit): fraction값과 곱해지는 지수 부분을 결정

- fraction(52bit): 실제 표현하는 값을 결정

표현되는 값의 공식: [[sign]] \* 1.[[fraction]]<sub>(2)</sub> \* 2<sup>[[exponent]] - 1023</sup> (1023은 정해진 bias값)

정수 13을 표현하려 할 때

- signbit는 양수이므로 '0<sub>(2)</sub>'이 된다.

- fraction 부분은 '1.1010000000....<sub>(2)</sub>', 1 소수점 밑에 101000...<sub>(2)</sub>의 52비트가 fraction이 된다. 맨앞에 1은 암묵적으로 추가된 1이다.

- 보정 값은 fraction부분을 3칸만큼 앞으로 당겨야 하므로 2<sup>3</sup> = 8이 되어야 한다. 따라서 exponent는 1026이 되어야 공식에 의해 2<sup>[[1026]]-bias(1023)</sup> = 8이 되므로 '10000000010<sub>(2)</sub>'이다.

결과적으로 13은 IEEE 754 double precision 스펙에 의해 0<sub>sign</sub>10000000010<sub>exponent</sub>10100000.....<sub>fraction(mantissa)</sub> 로 표현된다.

```js
//IEEE 754 표현법에 의해 fraction(mantissa) 부분의 52비트와 암묵적 1에 의해 2^53 - 1 범위까지 안전하게 표현할 수 있다.
// 2^53 - 1값을 넘기게 되면 fraction(mantissa) 부분의 정밀도를 넘어가기 때문에 규칙에 의하여 fraction(mantissa)값과 exponent값이 적절히 보정되어 표현할 수 있는 근사값을 표현한다. 따라서 오차가 발생한다.
const maxInteger = Number.MAX_SAFE_INTEGER; // 2^53 - 1
```

### Epsilon

JS 환경은 EPSILON 프로퍼티를 지원한다.

epsilon은 1과 1이상 다음의 가장 작은 소수간의 차이값을 나타낸다.

일반적으로 부동소수점의 허용가능한 오차 범위를 나타낼 때 사용한다.

```js
const isEqualFloat = (a, b) => {
	return Math.abs(a - b) < Number.EPSILON;
};
```

# String

```js
// 생성자 함수로써 호출된 String 메서드는 string wrapper object를 반환한다.
// 문자열 값은 [[StringData]] 내부슬롯에 저장된다. ES5에서는 [[StringData]] 내부 슬롯을 [[PrimitiveValue]] 내부 슬롯으로 설명한다.
const str = new String(123);

// string 객체는 각 문자에 개별 접근할 수 있는 인덱스 정수로써 프로퍼티와 length 프로퍼티를 포함하므로 유사 배열 객체(Array-like Object)이며 이터러블(iterable)이다.
// string 객체의 문자열 값은 원시값(primitive value) 이며, 변경할 수 없다. 다만 변경을 시도할 시 변경은 되지 않고 에러도 발생하지 않는다.
// string 객체의 각 문자값, 즉 프로퍼티의 프로퍼티 어트리뷰트 설정을 확인했을 때 수정 불가능 하도록 property attribute가 설정되어 있음을 알 수 있다.
```

## String prototype method

- String 프로토타입 메서드는 모두 원본값을 변경시키지 않고 새로운 값만 생성하는 **accessor method** 만 있다.

- 원본값 자체를 변경하는 **mutator method** 는 없다.

```js
const camelToSnake = (camelCase) => {
	// string.prototype.replace 메서드는 인수로 정규 표현식과 match 값을 제어하는 callback function을 입력할 수 있어 유용하다.
	return camelCase.replace(/.[A-Z]/g, (match) => {
		return match[0] + '_' + match[1].toLowerCase();
	});
};

const snakeToCamel = (snake_case) => {
	return snake_case.replace(/_[a-z]/g, (match) => {
		return match[1].toUpperCase();
	});
};
``;
```
