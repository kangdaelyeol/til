## 정보설계와 정보구조

### 정보설계(Information Design)

**`웹디자인 관련 정보 종류`**

| 종류               | 설명                                                    |
| ------------------ | ------------------------------------------------------- |
| 구체적 사실(Facts) | 설명 없이도 이해할 수 있는 구체적인 정보                |
| 개념(Concepts)     | 이해하기 쉽도록 정의된 정보                             |
| 절차(Procedures)   | 특정 작업을 수행하는 과정에 대한 정보                   |
| 과정(Processes)    | 특정 시스템 또는 아이템 작동 원리와 과정에 대한 정보    |
| 원칙(Principles)   | 사용자와 콘텐츠 제공자에게 요구되는 원칙 또는 행동 지침 |

**`웹디자인 정보 설계를 위한 기초 지식`**

| 종류                             | 설명                                                  |
| -------------------------------- | ----------------------------------------------------- |
| 웹사이트 정보구조 이해           | 정보구조를 이해하여 쉽게 정보를 탐색할 수 있도록 설계 |
| 내비게이션 구조 설계             | 사용자 친화적이면서, 정보 탐색이 용이하도록 설계      |
| 구축 기능 이해 및 사용 도구 운용 | 웹 구축 도구를 능숙하게 이용                          |

### 정보 체계화

정보 체계화는 개별적으로 분산된 정보(콘텐츠)들을 체계적으로 그룹화하는 과정이다.

**`정보체계화 방법`**

| 종류                 | 설명                                              |
| -------------------- | ------------------------------------------------- |
| 특징이 명확한 정보   | 알파벳순, 연대와 날짜, 위치나 장소 등에 따른 분류 |
| 특징이 불명확한 정보 | 주제, 기능, 이용자, 상징이나 의미 등에 따른 분류  |

**`정보체계화 주요 요소`**

| 종류               | 설명                                                                      |
| ------------------ | ------------------------------------------------------------------------- |
| 정보분류           | 카테고리, 서비스 유형, 사용자 그룹 등에 따른 분류                         |
| 정보구조화         | 정보의 계층구조 설계, 주요 메뉴와 하위 메뉴 구조 설계                     |
| 레이블링(Labeling) | 정보체계에 레이블을 붙여서 사용자가 혼동 없이 정보를 찾을 수 있게 하는 것 |
| 내비게이션         | 검색기능, 드롭다운 메뉴 등을 통해 쉽게 정보를 탐색할 수 있게 하는 것      |

### 정보구조(Information Architecture)

웹디자인에서 정보구조란, 정보와 콘텐츠, 기능 등을 체계적으로 조직하는 것이다.

**`정보구조 요소`**

| 종류            | 설명                                   |
| --------------- | -------------------------------------- |
| 계층(Hierarchy) | 정보의 상하위 관계                     |
| 깊이(Depth)     | 계층의 최대 깊이                       |
| 폭(Width)       | 특정 계층에서 선택할 수 있는 항목의 수 |
| 레벨(Level)     | 같은 깊이에 존재하는 항목              |

**`정보구조 유형`**

| 종류                                      | 설명                                                                                                                                       |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| 계층 구조(Hierarchy Structure)            | 정보가 상,하위 카테고리로 나뉘어 계층적으로 분류되는 구조                                                                                  |
| 선형 구조(Linear Structure)               | 정보가 정해진 순서대로 나열되어 처음부터 끝까지 순차적으로 탐색하는 구조. 튜토리얼, 온보딩 플로우 등에 사용                                |
| 허브 앤드 스포크(Hub and Spoke Structure) | 중앙 허브(Hub) 페이지를 기준으로 여러 개의 스포크(Spoke) 페이지로 분기되는 구조. 각 스포크 간 직접 이동은 불가하고 반드시 허브를 거쳐야 함 |
| 중첩 인형(Nested Doll Structure)          | 큰 개요 페이지부터 시작해 점점 세부 정보 페이지로 드릴다운(drill-down)하는 구조. 러시안 인형처럼 안으로 파고드는 계층 탐색                 |
| 네트워크 구조(Network/Web Structure)      | 모든 페이지가 링크를 통해 자유롭게 연결된 구조. 탐색 경로가 비선형적이며 백과사전, 위키 등에 사용                                          |
| 대시보드(Dashboard Structure)             | 홈 화면에서 다양한 정보를 한 눈에 볼 수 있도록 제공하는 구조                                                                               |
| 레이블링(Labeling Structure)              | 정보에 레이블을 붙여 사용자가 혼동 없이 정보를 식별할 수 있게 하는 구조                                                                    |

### 내비게이션

내비게이션이란 사용자가 다양한 섹션 및 페이지로 이동할 수 있는 링크들을 체계적으로 모아놓은 것이다.

**`내비게이션 구성요소`**

| 종류              | 설명                                                                              |
| ----------------- | --------------------------------------------------------------------------------- |
| 내비게이션바      | 웹페이지 상단에 위치하며 주요 메뉴 링크를 모아놓은 수평 영역                      |
| 메뉴              | 카테고리 및 페이지 이동을 위한 항목들의 목록                                      |
| 링크              | 클릭 시 다른 페이지나 섹션으로 이동하는 하이퍼텍스트 요소                         |
| 이미지맵          | 이미지의 특정 좌표 영역에 링크를 연결하여 클릭 시 해당 URL로 이동하는 방식        |
| 사이트맵          | 웹사이트의 전체 페이지 구조를 계층적으로 보여주는 페이지 또는 XML 파일            |
| 사이드바          | 메인 콘텐츠 옆에 위치한 보조 내비게이션 영역. 카테고리, 태그, 최근 게시물 등 제공 |
| 바닥글 내비게이션 | 페이지 하단 Footer에 위치한 보조 링크 모음. 이용약관, 회사소개 등 포함            |
| 기타              | 브레드크럼(Breadcrumb), 페이지네이션(Pagination), 탭(Tab) 등                      |

**`내비게이션 메뉴 종류`**

| 종류                    | 설명                                                                                       |
| ----------------------- | ------------------------------------------------------------------------------------------ |
| 고정된 헤더             | 스크롤해도 화면 상단에 고정(fixed/sticky)되어 항상 접근 가능한 내비게이션 바               |
| 드롭다운(Dropdown) 메뉴 | 메뉴 항목을 클릭하거나 호버하면 하위 메뉴가 아래로 펼쳐지는 방식                           |
| 햄버거(Hamburger) 메뉴  | 세 줄 아이콘(☰)을 클릭하면 숨겨진 메뉴가 슬라이드되어 표시되는 방식. 주로 모바일에서 사용 |
| 메가(Mega) 메뉴         | 드롭다운의 확장형으로, 다수의 카테고리와 링크를 여러 열에 걸쳐 한번에 보여주는 대형 메뉴   |

**`내비게이션 유형`**

| 종류                   | 설명                                                                                                |
| ---------------------- | --------------------------------------------------------------------------------------------------- |
| 순차(Sequence) 구조    | 이전/다음 버튼으로 정해진 순서대로 페이지를 이동하는 방식. 튜토리얼, 결제 플로우 등에 사용          |
| 그리드(Grid) 구조      | 동등한 위계의 항목들을 격자 형태로 배치하여 탐색하는 방식. 갤러리, 상품 목록 등에 사용              |
| 계층(Hierarchy) 구조   | 상위-하위 메뉴 체계로 이동하는 방식. 일반 웹사이트에서 가장 보편적으로 사용                         |
| 네트워크(Network) 구조 | 관련 콘텐츠를 링크로 자유롭게 연결하여 비선형적으로 탐색하는 방식. 위키, 검색 기반 사이트 등에 사용 |

## 와이어프레임 작성과 레이아웃 및 스토리보드

### 와이어프레임(Wireframe) 작성

와이어프레임이란, 웹페이지 구조와 레이아웃을 간단한 선과 사각형 정도만을 사용하여 윤곽을 그린 설계 도구를 의미한다.

전체적인 주요 UI 요소의 배치를 보여주기 위해 작성한다.

색상, Typography, 이미지 등 시각적 요소는 최소화하고, 화면의 계층구조와 주요 요소의 위치를 중심으로 표현한다.

### 레이아웃(Layout)

레이아웃이란, 콘텐츠를 적절하게 배치시킨 구조 또는 형태를 의미한다.

와이어프레임에서 초기 아이디어 및 구조에 중점을 두고, 이를 기반으로 레이아웃 설계 단계에서 시각적 요소 배치를 결정하여 웹페이지의 골격을 구체화한다.

**`레이아웃의 구성요소 및 기능`**

| 종류             | 설명                                                                    |
| ---------------- | ----------------------------------------------------------------------- |
| 헤더(Header)     | 페이지 상단 영역. 로고, 내비게이션 바, 검색창 등이 위치                 |
| 내비게이션(Nav)  | 페이지 간 이동을 위한 메뉴 링크 모음 영역                               |
| 콘텐츠(Contents) | 페이지의 핵심 정보가 위치하는 메인 영역                                 |
| 어사이드(Aside)  | 메인 콘텐츠 옆에 위치한 보조 정보 영역. 관련 링크, 광고, 태그 등이 위치 |
| 푸터(Footer)     | 페이지 하단 영역. 저작권, 이용약관, 연락처, 사이트맵 등이 위치          |
| 광고             | 배너 광고 또는 스폰서 콘텐츠가 배치되는 영역                            |
| 기타 영역 분할   | 위에 해당하지 않는 보조 영역. 모달, 팝업, 오버레이 등                   |

**`레이아웃 종류`**

| 종류                              | 특징                                                                                     | 단점                                                                           |
| --------------------------------- | ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| 고정형 너비(Fixed width) 레이아웃 | 콘텐츠 너비를 px 단위로 고정. 화면 크기와 무관하게 동일한 레이아웃 유지                  | 화면이 좁으면 가로 스크롤 발생, 넓으면 좌우 여백이 생겨 공간 낭비              |
| 유동형(Fluid) 레이아웃            | 콘텐츠 너비를 % 단위로 지정하여 화면 크기에 비례해 늘어나거나 줄어듦                     | 화면이 너무 넓거나 좁을 때 레이아웃이 과도하게 늘어나거나 압축되어 가독성 저하 |
| 반응형(Responsive) 레이아웃       | 미디어 쿼리(Media Query)로 브레이크포인트마다 레이아웃을 다르게 적용. 하나의 HTML로 대응 | 구현이 복잡하고 다양한 화면 크기를 대응하기 위한 CSS 작업량이 많음             |
| 적응형(Adaptive) 레이아웃         | 기기를 감지하여 해당 기기에 맞는 고정 레이아웃을 제공. 기기별 레이아웃을 미리 제작       | 기기별 별도 레이아웃을 유지해야 하므로 개발 및 유지보수 비용이 높음            |

### 스토리보드(Storyboard)

스토리보드란, 일종의 작업 지침서로서 웹사이트의 전체 구성을 표현한 문서이다.

화면에 대한 레이아웃, 내비게이션, 기능 등을 그림과 설명으로 시각화한다.

주로 와이어프레임에 시나리오를 포함하여 사용자가 웹사이트 각 단계를 어떻게 사용하게 될지를 표현한다.

디자인 요소보다는 페이지에 노출되는 주요 요소를 표현하는 것이 중요하다.

**`스토리보드 작성 내용`**

| 종류             | 설명                                                                    |
| ---------------- | ----------------------------------------------------------------------- |
| 표지 페이지      | 프로젝트명, 문서 버전, 작성일, 작성자 등 문서 기본 정보                 |
| 개정 이력        | 버전, 날짜, 수정 내용, 수정자를 기록한 문서 변경 이력                   |
| 화면 설계        | 각 페이지의 레이아웃, UI 요소 배치, 인터랙션을 시각적으로 표현한 설계도 |
| 서비스 흐름도    | 사용자가 서비스를 이용하는 전체 흐름을 플로우차트로 표현                |
| 페이지 상세 정보 | 각 화면의 URL, 기능명, 화면 설명, 연결 페이지 등 세부 명세              |
| 디자인 요소      | 색상, 폰트, 아이콘 등 시각적 디자인 가이드라인                          |
| 기능과 요구사항  | 각 화면에서 동작해야 하는 기능 목록 및 비기능 요구사항 정의             |

**`페르소나(Persona)`**

UX 디자인 과정에서 스토리보드는 사용자의 경험을 시각적으로 표현하는 도구다.

UX 디자인에서 가상의 사용자 프로필을 페르소나(Persona)라고 한다. 페르소나를 이용하여 스토리보드를 작성한다.

## 디자인의 의미와 조건

### 디자인 용어

디자인(Design)은 이탈(De)과 형상(Sign)의 합성어다.

Design은 '계획을 세우다', '지시하다', '스케치하다'라는 의미의 라틴어 '데시그나레(Designare)'로부터 유래되었다.

### 디자인 개념

디자인(Design)의 사전적 의미는 '의장', '도안', '계획', '설계'다.

**`의미 구분`**

| 구분      | 설명                                                            |
| --------- | --------------------------------------------------------------- |
| 좁은 의미 | 사용하기 쉽고 안전하며, 아름답고 쾌적한 생활 환경을 창조하는 것 |
| 넓은 의미 | 심적 계획(A mental plan)                                        |

**`빅터 파파넥(Victor Papanek)의 복합 기능(Function Complex)`**

빅터 파파넥은 디자인을 형태나 기능으로 정의하는 것을 넘어서, '방법, 용도, 필요, 목적(telesis), 연상, 미학'의 측면을 복합적으로 고려해야 한다는 복합 기능(Function Complex)이라는 포괄적인 개념을 제시하였다.

| 기능              | 설명                                                                     |
| ----------------- | ------------------------------------------------------------------------ |
| 방법(Method)      | 어떤 재료와 기술, 공정으로 만들어지는가. 제작 수단과 도구                |
| 용도(Use)         | 제품이 얼마나 제대로 기능하는가. 실제 사용성과 기능적 효율성             |
| 필요(Needs)       | 사용자의 신체적·심리적·사회적 필요와 욕구를 충족하는가                   |
| 목적(Telesis)     | 사회·문화적 목적에 부합하는가. 디자인이 사회 발전 방향에 기여하는지 여부 |
| 연상(Association) | 제품이 불러일으키는 심리적 연상과 감정. 문화적 맥락과 상징               |
| 미학(Aesthetics)  | 형태의 아름다움과 감각적 매력. 시각적·촉각적 심미 요소                   |

### 뉴미디어 디자인과 리디자인(Re-Design)

| 종류            | 설명                                                                                                 |
| --------------- | ---------------------------------------------------------------------------------------------------- |
| 뉴미디어 디자인 | 디지털 기술과 인터랙티브 미디어를 기반으로 한 디자인. 웹, 모바일, 멀티미디어 환경을 위한 시각 디자인 |
| 리디자인        | 기존 디자인을 사용자 피드백, 기술 변화, 트렌드에 맞게 개선하거나 재구성하는 것                       |

### 디자인의 조건

합목적성, 경제성, 심미성, 독창성을 디자인의 4대 조건이라 하고, 질서성까지 더해지면 5대 조건이 된다.

디자인의 5대 조건을 만족시킨 디자인을 **굿 디자인(Good Design)** 이라고 한다.

| 종류     | 설명                                                 |
| -------- | ---------------------------------------------------- |
| 합목적성 | 디자인이 그 목적과 기능에 부합해야 함. 기능적 적합성 |
| 경제성   | 최소의 재료와 노력으로 최대의 효과 달성. 비용 효율성 |
| 심미성   | 아름답고 감각적으로 만족스러워야 함. 시각적 매력     |
| 독창성   | 창의적이고 독자적인 특성을 가져야 함. 차별성과 혁신  |
| 질서성   | 통일성과 조화를 가져야 함. 균형과 일관성             |

### 디자인의 분류

**`시각 디자인(Visual Design)`**

시각 전달 디자인(Visual Communication Design) 또는 그래픽 디자인(Graphic Design)이라고도 한다.

신속하고 정확한 의미를 전달하는 특징이 있다.

**`Identity / Brand Identity`**

Identity란 기업의 본질, 특징, 가치를 포괄하는 근본적인 개념으로, 기업 통합 전략이다.

Brand Identity는 기업의 본질을 포착하고 전달하여 기업의 방향을 나타내는 것이다. 제품이 대중에게 전달하고자 하는 감각적 요소들의 총합이라고 볼 수 있다.

| 요소                       | 설명                                                                  |
| -------------------------- | --------------------------------------------------------------------- |
| 로고(Logo)                 | 브랜드를 상징하는 시각적 마크. 워드마크, 심볼마크, 조합형 등으로 구분 |
| 슬로건(Slogan)             | 브랜드의 핵심 가치나 메시지를 간결하게 표현한 문구                    |
| 캐릭터(Character)          | 브랜드를 대표하는 의인화된 또는 상징적 캐릭터                         |
| 컬러 팔레트(Color Palette) | 브랜드에서 일관되게 사용하는 색상 체계                                |

**`CI(Corporate Identity)`**

CI란 기업의 본질, 특징, 가치 등을 시각적으로 통합시켜 표현한 기업 정체성이다.

| 요소                 | 의미                                                           |
| -------------------- | -------------------------------------------------------------- |
| BI(Brand Identity)   | 제품·서비스 브랜드의 정체성을 시각적으로 표현한 체계           |
| MI(Mission Identity) | 기업의 철학, 비전, 미션 등 정신적 정체성                       |
| VI(Visual Identity)  | 로고, 색상, 서체 등 시각 요소를 통해 기업 정체성을 표현한 체계 |

**`CIP(Corporate Identity Program)`**

CIP는 CI를 유지하기 위한 프로그램이다.

| 종류          | 설명                                                                |
| ------------- | ------------------------------------------------------------------- |
| 베이직 시스템 | 로고, 심볼, 색상, 서체 등 CI의 기본 요소를 정의한 표준 규정         |
| 응용 시스템   | 베이직 시스템을 명함, 봉투, 유니폼, 간판 등 실제 매체에 적용한 규정 |

### 웹사이트 디자인 요소

| 요소                  | 설명                                                                                                            |
| --------------------- | --------------------------------------------------------------------------------------------------------------- |
| 배치 및 레이아웃      | 그리드 시스템, 여백(White Space), 정렬 등을 통해 콘텐츠를 체계적으로 구성                                       |

| 타이포그래피          | 서체, 글꼴 크기, 행간, 자간 등 텍스트 표현 요소. 가독성과 시각적 위계 구조를 결정                               |
| 이미지와 그래픽       | 콘텐츠를 시각적으로 보완하는 사진, 일러스트, 아이콘 등. 품질과 최적화도 고려                                    |
| 내비게이션            | 사용자가 페이지 간 이동하는 메뉴 구조. 직관적이고 일관된 내비게이션은 UX에 핵심적                               |

| 메타포(Metaphor)      | 실생활의 친숙한 개념을 디지털 인터페이스에 적용하여 사용자 이해를 돕는 방식. 쇼핑카트, 폴더 아이콘 등           |

### 웹사이트 조형 요소

조형 요소는 시각적이며 구조적인 구성 요소를 의미한다.

조형 요소는 크게 개념 요소, 시각 요소, 상관 요소로 구분된다.

**`개념 요소`**

개념 요소란 물리적으로 눈으로 볼 수 없지만, 인지와 이해를 통해 존재를 인식(지각)할 수 있는 요소를 말한다.

| 종류        | 설명                                                                                      |
| ----------- | ----------------------------------------------------------------------------------------- |
| 점(Point)   | 위치만 있고 크기가 없는 가장 기본적인 조형 요소. 집중·강조 효과                           |
| 선(Line)    | 점이 이동한 궤적. 방향성과 운동감을 표현. 수평선(안정감), 수직선(긴장감), 곡선(유연함) 등 |
| 면(Plane)   | 선이 이동한 궤적. 2차원적 형태로 넓이와 방향을 가짐                                       |
| 입체(Solid) | 면이 이동한 궤적. 3차원적 형태로 깊이·높이·너비를 가짐                                    |

선은 개념적으로 Positive / Negative Line으로 구분할 수 있다.

| 종류          | 설명                                                   |
| ------------- | ------------------------------------------------------ |
| Positive line | 실제로 그어진 선. 형태나 경계를 직접적으로 나타내는 선 |
| Negative line | 두 형태 사이의 공간·여백에 의해 암시적으로 생성되는 선 |

**`시각 요소`**

형태를 눈으로 지각할 수 있는 요소를 시각 요소라고 한다.

| 요소          | 설명                                                                           |
| ------------- | ------------------------------------------------------------------------------ |
| 형(Shape)     | 2차원적 외곽선으로 만들어진 윤곽. 평면적 형태                                  |
| 형태(Form)    | 3차원적 입체 모양. 깊이와 부피를 포함                                          |
| 크기(Size)    | 요소의 상대적 크기. 비례와 강조에 영향을 줌                                    |
| 색채(Color)   | 색조(Hue)·명도(Value)·채도(Chroma)로 구성. 감정과 인식에 영향을 주는 시각 요소 |
| 질감(Texture) | 표면의 시각적·촉각적 특성. 재질감을 표현                                       |

형태(Form)은 이념적 / 현실적 형태로 구분할 수 있다.

| 종류              | 설명                                                                             |
| ----------------- | -------------------------------------------------------------------------------- |
| 이념적 형태       | 자연에 실재하지 않는 순수 추상 형태. 원, 삼각형 등 기하학적 형태                 |
| 현실적(실제) 형태 | 자연이나 현실에 실제로 존재하는 형태. 자연 형태(식물·동물 등)와 인공 형태로 구분 |

질감은 촉각적 / 시각적 질감으로 구분할 수 있다.

| 종류        | 설명                                                                                           |
| ----------- | ---------------------------------------------------------------------------------------------- |
| 촉각적 질감 | 실제로 손으로 만졌을 때 느낄 수 있는 표면의 물리적 특성                                        |
| 시각적 질감 | 눈으로 보았을 때 질감처럼 느껴지는 시각적 효과. 실제로 만질 수 없음 (예: 화면의 텍스처 이미지) |

**`상관 요소`**

상관 요소란, 개념 요소와 시각 요소 등 디자인 요소들의 결합을 통해 나타나는 부차적 속성이다.

| 종류   | 설명                                                                           |
| ------ | ------------------------------------------------------------------------------ |
| 위치   | 요소가 화면 내에서 차지하는 공간적 위치. 시선의 흐름과 강조에 영향             |
| 방향   | 요소가 가리키거나 흐르는 방향감. 수평·수직·사선 등                             |
| 공간감 | 2차원 평면에서 3차원적 깊이를 느끼게 하는 효과. 원근감, 겹침, 크기 차이로 표현 |
| 중량감 | 형태·색상·크기의 조합으로 느껴지는 시각적 무게감. 균형에 영향                  |
