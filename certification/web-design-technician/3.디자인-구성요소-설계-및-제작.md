## 정보설계와 정보구조

### 정보설계(Information Design)

**`웹디자인 관련 정보 종류`**

| 종류               | 설명                                                    |
| ------------------ | ------------------------------------------------------- |
| 구체적 사실(Facts) | 설명 없이도 이해할 수 있는 구체적인 정보                |
| 개념(Concepts)     | 이해하기 쉽도록 정의된 정보                             |
| 절차(Procedures)   | 특정 작업을 수행하는 과정에 대한 정보                   |
| 과정(Processes)    | 특정 시스템 또는 아이템 작동 원리와 과정에 대한 정보    |
| 원칙(Principles)   | 사용자와 콘텐츠 제공자에게 요구되는 원칙 또는 행동 지침 |

**`웹디자인 정보 설계를 위한 기초 지식`**

| 종류                             | 설명                                                  |
| -------------------------------- | ----------------------------------------------------- |
| 웹사이트 정보구조 이해           | 정보구조를 이해하여 쉽게 정보를 탐색할 수 있도록 설계 |
| 내비게이션 구조 설계             | 사용자 친화적이면서, 정보 탐색이 용이하도록 설계      |
| 구축 기능 이해 및 사용 도구 운용 | 웹 구축 도구를 능숙하게 이용                          |

### 정보 체계화

정보 체계화는 개별적으로 분산된 정보(콘텐츠)들을 체계적으로 그룹화하는 과정이다.

**`정보체계화 방법`**

| 종류                 | 설명                                              |
| -------------------- | ------------------------------------------------- |
| 특징이 명확한 정보   | 알파벳순, 연대와 날짜, 위치나 장소 등에 따른 분류 |
| 특징이 불명확한 정보 | 주제, 기능, 이용자, 상징이나 의미 등에 따른 분류  |

**`정보체계화 주요 요소`**

| 종류               | 설명                                                                      |
| ------------------ | ------------------------------------------------------------------------- |
| 정보분류           | 카테고리, 서비스 유형, 사용자 그룹 등에 따른 분류                         |
| 정보구조화         | 정보의 계층구조 설계, 주요 메뉴와 하위 메뉴 구조 설계                     |
| 레이블링(Labeling) | 정보체계에 레이블을 붙여서 사용자가 혼동 없이 정보를 찾을 수 있게 하는 것 |
| 내비게이션         | 검색기능, 드롭다운 메뉴 등을 통해 쉽게 정보를 탐색할 수 있게 하는 것      |

### 정보구조(Information Architecture)

웹디자인에서 정보구조란, 정보와 콘텐츠, 기능 등을 체계적으로 조직하는 것이다.

**`정보구조 요소`**

| 종류            | 설명                                   |
| --------------- | -------------------------------------- |
| 계층(Hierarchy) | 정보의 상하위 관계                     |
| 깊이(Depth)     | 계층의 최대 깊이                       |
| 폭(Width)       | 특정 계층에서 선택할 수 있는 항목의 수 |
| 레벨(Level)     | 같은 깊이에 존재하는 항목              |

**`정보구조 유형`**

| 종류                                      | 설명                                                                                                                                       |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| 계층 구조(Hierarchy Structure)            | 정보가 상,하위 카테고리로 나뉘어 계층적으로 분류되는 구조                                                                                  |
| 선형 구조(Linear Structure)               | 정보가 정해진 순서대로 나열되어 처음부터 끝까지 순차적으로 탐색하는 구조. 튜토리얼, 온보딩 플로우 등에 사용                                |
| 허브 앤드 스포크(Hub and Spoke Structure) | 중앙 허브(Hub) 페이지를 기준으로 여러 개의 스포크(Spoke) 페이지로 분기되는 구조. 각 스포크 간 직접 이동은 불가하고 반드시 허브를 거쳐야 함 |
| 중첩 인형(Nested Doll Structure)          | 큰 개요 페이지부터 시작해 점점 세부 정보 페이지로 드릴다운(drill-down)하는 구조. 러시안 인형처럼 안으로 파고드는 계층 탐색                 |
| 네트워크 구조(Network/Web Structure)      | 모든 페이지가 링크를 통해 자유롭게 연결된 구조. 탐색 경로가 비선형적이며 백과사전, 위키 등에 사용                                          |
| 대시보드(Dashboard Structure)             | 홈 화면에서 다양한 정보를 한 눈에 볼 수 있도록 제공하는 구조                                                                               |
| 레이블링(Labeling Structure)              | 정보에 레이블을 붙여 사용자가 혼동 없이 정보를 식별할 수 있게 하는 구조                                                                    |

### 내비게이션

내비게이션이란 사용자가 다양한 섹션 및 페이지로 이동할 수 있는 링크들을 체계적으로 모아놓은 것이다.

**`내비게이션 구성요소`**

| 종류              | 설명                                                                              |
| ----------------- | --------------------------------------------------------------------------------- |
| 내비게이션바      | 웹페이지 상단에 위치하며 주요 메뉴 링크를 모아놓은 수평 영역                      |
| 메뉴              | 카테고리 및 페이지 이동을 위한 항목들의 목록                                      |
| 링크              | 클릭 시 다른 페이지나 섹션으로 이동하는 하이퍼텍스트 요소                         |
| 이미지맵          | 이미지의 특정 좌표 영역에 링크를 연결하여 클릭 시 해당 URL로 이동하는 방식        |
| 사이트맵          | 웹사이트의 전체 페이지 구조를 계층적으로 보여주는 페이지 또는 XML 파일            |
| 사이드바          | 메인 콘텐츠 옆에 위치한 보조 내비게이션 영역. 카테고리, 태그, 최근 게시물 등 제공 |
| 바닥글 내비게이션 | 페이지 하단 Footer에 위치한 보조 링크 모음. 이용약관, 회사소개 등 포함            |
| 기타              | 브레드크럼(Breadcrumb), 페이지네이션(Pagination), 탭(Tab) 등                      |

**`내비게이션 메뉴 종류`**

| 종류                    | 설명                                                                                       |
| ----------------------- | ------------------------------------------------------------------------------------------ |
| 고정된 헤더             | 스크롤해도 화면 상단에 고정(fixed/sticky)되어 항상 접근 가능한 내비게이션 바               |
| 드롭다운(Dropdown) 메뉴 | 메뉴 항목을 클릭하거나 호버하면 하위 메뉴가 아래로 펼쳐지는 방식                           |
| 햄버거(Hamburger) 메뉴  | 세 줄 아이콘(☰)을 클릭하면 숨겨진 메뉴가 슬라이드되어 표시되는 방식. 주로 모바일에서 사용 |
| 메가(Mega) 메뉴         | 드롭다운의 확장형으로, 다수의 카테고리와 링크를 여러 열에 걸쳐 한번에 보여주는 대형 메뉴   |

**`내비게이션 유형`**

| 종류                   | 설명                                                                                                |
| ---------------------- | --------------------------------------------------------------------------------------------------- |
| 순차(Sequence) 구조    | 이전/다음 버튼으로 정해진 순서대로 페이지를 이동하는 방식. 튜토리얼, 결제 플로우 등에 사용          |
| 그리드(Grid) 구조      | 동등한 위계의 항목들을 격자 형태로 배치하여 탐색하는 방식. 갤러리, 상품 목록 등에 사용              |
| 계층(Hierarchy) 구조   | 상위-하위 메뉴 체계로 이동하는 방식. 일반 웹사이트에서 가장 보편적으로 사용                         |
| 네트워크(Network) 구조 | 관련 콘텐츠를 링크로 자유롭게 연결하여 비선형적으로 탐색하는 방식. 위키, 검색 기반 사이트 등에 사용 |

## 와이어프레임 작성과 레이아웃 및 스토리보드

### 와이어프레임(Wireframe) 작성

와이어프레임이란, 웹페이지 구조와 레이아웃을 간단한 선과 사각형 정도만을 사용하여 윤곽을 그린 설계 도구를 의미한다.

전체적인 주요 UI 요소의 배치를 보여주기 위해 작성한다.

색상, Typography, 이미지 등 시각적 요소는 최소화하고, 화면의 계층구조와 주요 요소의 위치를 중심으로 표현한다.

### 레이아웃(Layout)

레이아웃이란, 콘텐츠를 적절하게 배치시킨 구조 또는 형태를 의미한다.

와이어프레임에서 초기 아이디어 및 구조에 중점을 두고, 이를 기반으로 레이아웃 설계 단계에서 시각적 요소 배치를 결정하여 웹페이지의 골격을 구체화한다.

**`레이아웃의 구성요소 및 기능`**

| 종류             | 설명                                                                    |
| ---------------- | ----------------------------------------------------------------------- |
| 헤더(Header)     | 페이지 상단 영역. 로고, 내비게이션 바, 검색창 등이 위치                 |
| 내비게이션(Nav)  | 페이지 간 이동을 위한 메뉴 링크 모음 영역                               |
| 콘텐츠(Contents) | 페이지의 핵심 정보가 위치하는 메인 영역                                 |
| 어사이드(Aside)  | 메인 콘텐츠 옆에 위치한 보조 정보 영역. 관련 링크, 광고, 태그 등이 위치 |
| 푸터(Footer)     | 페이지 하단 영역. 저작권, 이용약관, 연락처, 사이트맵 등이 위치          |
| 광고             | 배너 광고 또는 스폰서 콘텐츠가 배치되는 영역                            |
| 기타 영역 분할   | 위에 해당하지 않는 보조 영역. 모달, 팝업, 오버레이 등                   |

**`레이아웃 종류`**

| 종류                              | 특징                                                                                     | 단점                                                                           |
| --------------------------------- | ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| 고정형 너비(Fixed width) 레이아웃 | 콘텐츠 너비를 px 단위로 고정. 화면 크기와 무관하게 동일한 레이아웃 유지                  | 화면이 좁으면 가로 스크롤 발생, 넓으면 좌우 여백이 생겨 공간 낭비              |
| 유동형(Fluid) 레이아웃            | 콘텐츠 너비를 % 단위로 지정하여 화면 크기에 비례해 늘어나거나 줄어듦                     | 화면이 너무 넓거나 좁을 때 레이아웃이 과도하게 늘어나거나 압축되어 가독성 저하 |
| 반응형(Responsive) 레이아웃       | 미디어 쿼리(Media Query)로 브레이크포인트마다 레이아웃을 다르게 적용. 하나의 HTML로 대응 | 구현이 복잡하고 다양한 화면 크기를 대응하기 위한 CSS 작업량이 많음             |
| 적응형(Adaptive) 레이아웃         | 기기를 감지하여 해당 기기에 맞는 고정 레이아웃을 제공. 기기별 레이아웃을 미리 제작       | 기기별 별도 레이아웃을 유지해야 하므로 개발 및 유지보수 비용이 높음            |

### 스토리보드(Storyboard)

스토리보드란, 일종의 작업 지침서로서 웹사이트의 전체 구성을 표현한 문서이다.

화면에 대한 레이아웃, 내비게이션, 기능 등을 그림과 설명으로 시각화한다.

주로 와이어프레임에 시나리오를 포함하여 사용자가 웹사이트 각 단계를 어떻게 사용하게 될지를 표현한다.

디자인 요소보다는 페이지에 노출되는 주요 요소를 표현하는 것이 중요하다.

**`스토리보드 작성 내용`**

| 종류             | 설명                                                                    |
| ---------------- | ----------------------------------------------------------------------- |
| 표지 페이지      | 프로젝트명, 문서 버전, 작성일, 작성자 등 문서 기본 정보                 |
| 개정 이력        | 버전, 날짜, 수정 내용, 수정자를 기록한 문서 변경 이력                   |
| 화면 설계        | 각 페이지의 레이아웃, UI 요소 배치, 인터랙션을 시각적으로 표현한 설계도 |
| 서비스 흐름도    | 사용자가 서비스를 이용하는 전체 흐름을 플로우차트로 표현                |
| 페이지 상세 정보 | 각 화면의 URL, 기능명, 화면 설명, 연결 페이지 등 세부 명세              |
| 디자인 요소      | 색상, 폰트, 아이콘 등 시각적 디자인 가이드라인                          |
| 기능과 요구사항  | 각 화면에서 동작해야 하는 기능 목록 및 비기능 요구사항 정의             |

**`페르소나(Persona)`**

UX 디자인 과정에서 스토리보드는 사용자의 경험을 시각적으로 표현하는 도구다.

UX 디자인에서 가상의 사용자 프로필을 페르소나(Persona)라고 한다. 페르소나를 이용하여 스토리보드를 작성한다.

## 디자인의 의미와 조건

### 디자인 용어

디자인(Design)은 이탈(De)과 형상(Sign)의 합성어다.

Design은 '계획을 세우다', '지시하다', '스케치하다'라는 의미의 라틴어 '데시그나레(Designare)'로부터 유래되었다.

### 디자인 개념

디자인(Design)의 사전적 의미는 '의장', '도안', '계획', '설계'다.

**`의미 구분`**

| 구분        | 설명                                                            |
| ----------- | --------------------------------------------------------------- |
| 좁은 의미   | 사용하기 쉽고 안전하며, 아름답고 쾌적한 생활 환경을 창조하는 것 |
| 넓은 의미   | 심적 계획(A mental plan)                                        |
| 사전적 의미 | 의장(意匠), 도안, 계획, 설계                                    |
| 본질적 의미 | 목적 달성을 위해 형태·기능·구조를 계획하는 창조적 문제해결 행위 |

**`빅터 파파넥(Victor Papanek)의 복합 기능(Function Complex)`**

빅터 파파넥은 디자인을 형태나 기능으로 정의하는 것을 넘어서, '방법, 용도, 필요, 목적(telesis), 연상, 미학'의 측면을 복합적으로 고려해야 한다는 복합 기능(Function Complex)이라는 포괄적인 개념을 제시하였다.

| 기능              | 설명                                                                     |
| ----------------- | ------------------------------------------------------------------------ |
| 방법(Method)      | 어떤 재료와 기술, 공정으로 만들어지는가. 제작 수단과 도구                |
| 용도(Use)         | 제품이 얼마나 제대로 기능하는가. 실제 사용성과 기능적 효율성             |
| 필요(Needs)       | 사용자의 신체적·심리적·사회적 필요와 욕구를 충족하는가                   |
| 목적(Telesis)     | 사회·문화적 목적에 부합하는가. 디자인이 사회 발전 방향에 기여하는지 여부 |
| 연상(Association) | 제품이 불러일으키는 심리적 연상과 감정. 문화적 맥락과 상징               |
| 미학(Aesthetics)  | 형태의 아름다움과 감각적 매력. 시각적·촉각적 심미 요소                   |

### 뉴미디어 디자인과 리디자인(Re-Design)

| 종류            | 설명                                                                                                 |
| --------------- | ---------------------------------------------------------------------------------------------------- |
| 뉴미디어 디자인 | 디지털 기술과 인터랙티브 미디어를 기반으로 한 디자인. 웹, 모바일, 멀티미디어 환경을 위한 시각 디자인 |
| 리디자인        | 기존 디자인을 사용자 피드백, 기술 변화, 트렌드에 맞게 개선하거나 재구성하는 것                       |

### 디자인의 조건

합목적성, 경제성, 심미성, 독창성을 디자인의 4대 조건이라 하고, 질서성까지 더해지면 5대 조건이 된다.

디자인의 5대 조건을 만족시킨 디자인을 **굿 디자인(Good Design)** 이라고 한다.

| 종류     | 설명                                                 |
| -------- | ---------------------------------------------------- |
| 합목적성 | 디자인이 그 목적과 기능에 부합해야 함. 기능적 적합성 |
| 경제성   | 최소의 재료와 노력으로 최대의 효과 달성. 비용 효율성 |
| 심미성   | 아름답고 감각적으로 만족스러워야 함. 시각적 매력     |
| 독창성   | 창의적이고 독자적인 특성을 가져야 함. 차별성과 혁신  |
| 질서성   | 통일성과 조화를 가져야 함. 균형과 일관성             |

### 디자인의 분류

**`시각 디자인(Visual Design)`**

시각 전달 디자인(Visual Communication Design) 또는 그래픽 디자인(Graphic Design)이라고도 한다.

신속하고 정확한 의미를 전달하는 특징이 있다.

**`Identity / Brand Identity`**

Identity란 기업의 본질, 특징, 가치를 포괄하는 근본적인 개념으로, 기업 통합 전략이다.

Brand Identity는 기업의 본질을 포착하고 전달하여 기업의 방향을 나타내는 것이다. 제품이 대중에게 전달하고자 하는 감각적 요소들의 총합이라고 볼 수 있다.

| 요소                       | 설명                                                                  |
| -------------------------- | --------------------------------------------------------------------- |
| 로고(Logo)                 | 브랜드를 상징하는 시각적 마크. 워드마크, 심볼마크, 조합형 등으로 구분 |
| 슬로건(Slogan)             | 브랜드의 핵심 가치나 메시지를 간결하게 표현한 문구                    |
| 캐릭터(Character)          | 브랜드를 대표하는 의인화된 또는 상징적 캐릭터                         |
| 컬러 팔레트(Color Palette) | 브랜드에서 일관되게 사용하는 색상 체계                                |

**`CI(Corporate Identity)`**

CI란 기업의 본질, 특징, 가치 등을 시각적으로 통합시켜 표현한 기업 정체성이다.

| 요소                 | 의미                                                           |
| -------------------- | -------------------------------------------------------------- |
| BI(Brand Identity)   | 제품·서비스 브랜드의 정체성을 시각적으로 표현한 체계           |
| MI(Mission Identity) | 기업의 철학, 비전, 미션 등 정신적 정체성                       |
| VI(Visual Identity)  | 로고, 색상, 서체 등 시각 요소를 통해 기업 정체성을 표현한 체계 |

**`CIP(Corporate Identity Program)`**

CIP는 CI를 유지하기 위한 프로그램이다.

| 종류          | 설명                                                                |
| ------------- | ------------------------------------------------------------------- |
| 베이직 시스템 | 로고, 심볼, 색상, 서체 등 CI의 기본 요소를 정의한 표준 규정         |
| 응용 시스템   | 베이직 시스템을 명함, 봉투, 유니폼, 간판 등 실제 매체에 적용한 규정 |

### 웹사이트 조형 요소

조형 요소는 시각적이며 구조적인 구성 요소를 의미한다.

조형 요소는 크게 개념 요소, 시각 요소, 상관 요소로 구분된다.

### 개념 요소

개념 요소란 물리적으로 눈으로 볼 수 없지만, 인지와 이해를 통해 존재를 인식(지각)할 수 있는 요소를 말한다.

| 종류        | 설명                                                                                      |
| ----------- | ----------------------------------------------------------------------------------------- |
| 점(Point)   | 위치만 있고 크기가 없는 가장 기본적인 조형 요소. 집중·강조 효과                           |
| 선(Line)    | 점이 이동한 궤적. 방향성과 운동감을 표현. 수평선(안정감), 수직선(긴장감), 곡선(유연함) 등 |
| 면(Plane)   | 선이 이동한 궤적. 2차원적 형태로 넓이와 방향을 가짐                                       |
| 입체(Solid) | 면이 이동한 궤적. 3차원적 형태로 깊이·높이·너비를 가짐                                    |

각 요소는 개념적으로 Positive / Negative 성격으로 구분할 수 있다.

**`선의 구분`**

| 종류                       | 설명                                                   |
| -------------------------- | ------------------------------------------------------ |
| 적극적인 선(Positive line) | 실제로 그어진 선. 형태나 경계를 직접적으로 나타내는 선 |
| 소극적인 선(Negative line) | 두 형태 사이의 공간·여백에 의해 암시적으로 생성되는 선 |

**`면의 구분`**

| 종류                        | 설명                                           |
| --------------------------- | ---------------------------------------------- |
| 적극적인 면(Positive Plane) | 형태나 색채로 채워진 실제로 보이는 면          |
| 소극적인 면(Negative Plane) | 형태 사이의 비어있는 공간·여백으로 인식되는 면 |

**`입체의 구분`**

| 종류                          | 설명                                                        |
| ----------------------------- | ----------------------------------------------------------- |
| 적극적인 입체(Positive Solid) | 현실적인 형으로 재료로 채워진 물리적으로 존재하는 입체 형태 |
| 소극적인 입체(Negative Solid) | 이념적인 형으로 크기, 폭 등이 없는 지각될 수 없는 형        |

### 시각 요소

형태를 눈으로 지각할 수 있는 요소를 시각 요소라고 한다.

| 요소          | 설명                                                                           |
| ------------- | ------------------------------------------------------------------------------ |
| 형(Shape)     | 2차원적 외곽선으로 만들어진 윤곽. 평면적 형태(2차원적 인식)                    |
| 형태(Form)    | 3차원적 입체 모양. 깊이와 부피를 포함(3차원적 인식)                            |
| 크기(Size)    | 요소의 상대적 크기. 비례와 강조에 영향을 줌                                    |
| 색채(Color)   | 색조(Hue)·명도(Value)·채도(Chroma)로 구성. 감정과 인식에 영향을 주는 시각 요소 |
| 질감(Texture) | 표면의 시각적·촉각적 특성. 재질감을 표현                                       |

**`형태(Form) 구분`**

| 종류              | 설명                                                                             |
| ----------------- | -------------------------------------------------------------------------------- |
| 이념적 형태       | 자연에 실재하지 않는 순수 추상 형태. 원, 삼각형 등 개념요소의 기하학적 형태      |
| 현실적(실제) 형태 | 자연이나 현실에 실제로 존재하는 형태. 자연 형태(식물·동물 등)와 인공 형태로 구분 |

**`질감(Texture) 구분`**

| 종류        | 설명                                                                              |
| ----------- | --------------------------------------------------------------------------------- |
| 촉각적 질감 | 실제로 손으로 만졌을 때 느낄 수 있는 표면의 물리적 특성(3차원적 질감)             |
| 시각적 질감 | 눈으로 보았을 때 질감처럼 느껴지는 시각적 효과. 실제로 만질 수 없음(2차원적 질감) |

### 상관 요소

상관 요소란, 개념 요소와 시각 요소 등 디자인 요소들의 결합을 통해 나타나는 부차적 속성이다.

| 종류   | 설명                                                                           |
| ------ | ------------------------------------------------------------------------------ |
| 위치   | 요소가 화면 내에서 차지하는 공간적 위치. 시선의 흐름과 강조에 영향             |
| 방향   | 요소가 가리키거나 흐르는 방향감. 수평·수직·사선 등                             |
| 공간감 | 2차원 평면에서 3차원적 깊이를 느끼게 하는 효과. 원근감, 겹침, 크기 차이로 표현 |
| 중량감 | 형태·색상·크기의 조합으로 느껴지는 시각적 무게감. 균형에 영향                  |

### 웹사이트 그래픽 요소

그래픽 요소는 시각적인 심미성을 높이고 정보 전달을 효과적으로 돕기 위해 사용한다.

| 요소                | 설명                                                                      |
| ------------------- | ------------------------------------------------------------------------- |
| 그래픽 이미지       | 사진, 일러스트레이션 등 시각적으로 정보를 전달하는 이미지 요소            |
| 아이콘(Icon)        | 기능이나 개념을 간결하게 나타낸 작은 그래픽 기호. 직관적인 UI 구성에 활용 |
| 픽토그램(Pictogram) | 사물·행동·개념을 단순화한 그림 문자. 언어와 무관하게 의미를 전달          |

### 디자인 원리

디자인 원리란 디자인 요소를 효과적으로 배치하고 구성하여 사용자에게 심미적이고 기능적인 경험을 제공하는 개념을 의미한다.

| 원리             | 특징                                                                                                               | 웹페이지에 적용                                             |
| ---------------- | ------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------- |
| 균형(Balance)    | 시각 요소들이 화면에서 시각적 무게감의 안정감을 이루는 원리. 대칭(Symmetry) 균형과 비대칭(Asymmetry) 균형으로 구분 | 좌우 레이아웃 배치, 콘텐츠와 여백의 균형                    |
| 조화(Harmony)    | 각 요소들이 서로 어울려 전체적으로 통일된 느낌을 주는 원리                                                         | 색상 팔레트, 폰트 계열, 이미지 스타일의 일관성 유지         |
| 대비(Contrast)   | 두 요소 간의 차이를 강조하여 시각적 명확성을 만드는 원리                                                           | 배경과 텍스트 색상 대비, 크기 차이, CTA 버튼 강조           |
| 강조(Emphasis)   | 특정 요소를 두드러지게 하여 시선을 집중시키는 원리                                                                 | 핵심 버튼, 헤드라인, 중요 이미지를 시각적으로 부각          |
| 리듬(Rhythm)     | 요소의 반복·간격·패턴으로 시각적 흐름과 움직임의 느낌을 만드는 원리                                                | 카드 목록, 반복되는 섹션 간격, 애니메이션 타이밍            |
| 비례(Proportion) | 요소들 간의 크기·양의 관계로 아름다움과 위계를 만드는 원리                                                         | 헤딩 크기 계층(h1>h2>h3), 콘텐츠 영역과 사이드바 비율       |
| 통일(Unity)      | 여러 요소들이 하나의 일관된 전체로 느껴지도록 하는 원리                                                            | 동일한 색상 테마, 폰트 시스템, 컴포넌트 스타일 일관 적용    |
| 동세(Movement)   | 시선이 자연스럽게 이동하도록 유도하는 원리. 선의 방향, 요소 배치 등으로 구현                                       | 시선 흐름을 유도하는 레이아웃, 스크롤 애니메이션, 화살표 등 |

**`균형(Balance) 구분`**

| 종류                    | 설명                                                                                          |
| ----------------------- | --------------------------------------------------------------------------------------------- |
| 대칭(Symmetric) 균형    | 중심축을 기준으로 좌우·상하가 동일하게 배치되어 안정감과 질서를 주는 균형. 공식적·정적인 느낌 |
| 비대칭(Asymmetric) 균형 | 서로 다른 요소들이 시각적 무게를 달리하여 균형을 이루는 방식. 역동적이고 생동감 있는 느낌     |

### 게슈탈트 법칙(Gestalt Principles)

게슈탈트 법칙에 의하면 형태는 근접, 유사, 폐쇄, 연속된 속성 등을 가질 때 하나의 그룹으로 인식되고 심리적으로 안정된 느낌을 준다.

| 법칙             | 설명                                                                     |
| ---------------- | ------------------------------------------------------------------------ |
| 근접성의 법칙    | 서로 가까이 있는 요소들을 하나의 그룹으로 인식하는 경향                  |
| 유사성의 법칙    | 형태·색상·크기 등이 유사한 요소들을 같은 그룹으로 인식하는 경향          |
| 연속성의 법칙    | 요소들이 연속적인 선이나 곡선을 이룰 때 하나의 흐름으로 인식하는 경향    |
| 폐쇄성의 법칙    | 불완전한 형태도 뇌가 자동으로 완성하여 완전한 형태로 인식하려는 경향     |
| 대칭성의 법칙    | 대칭적인 요소들을 하나의 단위로 인식하고 안정감을 느끼는 경향            |
| 전경-배경의 법칙 | 시각 장면을 주된 전경(Figure)과 배경(Ground)으로 분리하여 인식하는 경향  |
| 공통 운명의 법칙 | 같은 방향으로 움직이거나 변화하는 요소들을 하나의 그룹으로 인식하는 경향 |

### 착시(Optical Illusion)

착시(Optical Illusion)란 사물을 원래와 다르게 지각하는 시각적인 착오로, 과거의 경험 또는 심리적 배경에 기반하여 착각을 일으키게 되는 것이다.

| 종류                    | 설명                                                                                     |
| ----------------------- | ---------------------------------------------------------------------------------------- |
| 반전과 명도에 의한 착시 | 명도 차이나 색의 반전으로 인해 형태나 색이 원래와 다르게 보이는 현상                     |
| 각도와 방향의 착시      | 주변 선의 각도·방향 영향으로 실제로는 평행한 직선이 기울어져 보이는 현상                 |
| 크기의 착시             | 주변 요소의 크기 차이로 인해 같은 크기의 도형이 다르게 보이는 현상 (예: 에빙하우스 착시) |
| 속도의 착시             | 속도로 인해 요소가 짧거나 길게 보이는 착시                                               |
| 길이의 착시             | 화살표 방향 등의 영향으로 같은 길이의 선분이 다르게 보이는 현상 (예: 뮬러-라이어 착시)   |
| 색채와 명암 대비의 착시 | 주변 색이나 명암에 의해 동일한 색이 다른 색으로 인식되는 현상                            |
| 상방 거리 과대 착시     | 같은 크기 도형을 수직 정렬할 경우 위쪽 도형이 더 크게 보이는 착시                        |
| 수평-수직의 착시        | 같은 길이의 선이라도 수직선이 수평선보다 더 길어 보이는 현상                             |
| 반전 실체의 착시        | 평면적인 도형의 특정 부분을 튀어나오거나 들어간 입체적인 형태로 잘못 지각하는 현상       |

**`루빈의 법칙(Rubin's law)`**

루빈의 항아리(Rubin's Vase) 또는 **도형(전경)-배경 이론(Figure-Ground Theory)**으로 더 잘 알려져 있다.

형태는 바탕보다 더욱 잘 기억되며, 바탕은 형태보다 뒤에 있는 것 처럼 느껴지는 원리다

윤곽은 그 내부를 형태로 만들고 와부와 형태를 구분한다.

## 그리드 시스템 및 반응형 디자인

### 그리드 시스템(Grid system)

그리드 시스템이란 수직, 수평의 가상 격자로 구성된 가이드라인으로, 디자인 요소들을 정돈되고 균형 있게 배치하기 위해 사용된다.

**`그리드 시스템 구성요소`**

| 요소                | 설명                                                        |
| ------------------- | ----------------------------------------------------------- |
| 컨테이너(Container) | 그리드 전체를 감싸는 최상위 영역. 콘텐츠의 최대 너비를 제한 |
| 컬럼(Column)        | 콘텐츠를 배치하는 수직 구획. 일반적으로 12개 컬럼으로 구성  |
| 거터(Gutter)        | 컬럼 사이의 간격. 콘텐츠 간 여백을 만들어 가독성을 높임     |
| 마진(Margin)        | 그리드 좌우 바깥쪽 여백. 콘텐츠와 화면 가장자리 사이의 공간 |
| 모듈(Module)        | 컬럼과 행이 교차하는 단위 셀. 콘텐츠 배치의 최소 단위       |

### 반응형 디자인(Responsive Design)

반응형 디자인이란 웹 또는 앱이 다양한 장치와 화면 크기에서 최적의 사용자 경험을 제공할 수 있도록 디자인을 설계하는 것을 말한다.

**`반응형 디자인 주요 기술`**

| 종류                           | 설명                                                                                       |
| ------------------------------ | ------------------------------------------------------------------------------------------ |
| 반응형 그리드, 반응형 레이아웃 | Flexbox, Grid 등을 활용하여 화면 크기에 맞게 유연하게 변하는 레이아웃 구성                 |
| 뷰포트(Viewport) 설정          | `<meta name="viewport">` 태그로 디바이스 화면 너비에 맞게 렌더링되도록 설정                |
| Media query 사용               | 화면 너비·해상도 등의 조건에 따라 다른 CSS 스타일을 적용                                   |
| 유연한 이미지 및 미디어        | `max-width: 100%` 등을 사용하여 이미지가 컨테이너를 벗어나지 않도록 처리                   |
| 반응형 타이포그래피            | rem, em, vw 단위 등을 활용하여 화면 크기에 맞게 폰트 크기가 조절되도록 설정                |
| 로딩 속도 최적화               | 이미지 압축, 지연 로딩(Lazy Loading), 코드 최소화 등을 통해 다양한 환경에서 빠른 로딩 유지 |

### 반응형 레이아웃

반응형 레이아웃이란 다양한 화면 크기와 디바이스 환경에 맞추어 웹페이지 크기가 자동으로 최적화되는 기술이다.

Luke Wroblewski는 반응형 레이아웃 디자인 패턴을 다섯 가지로 구분한다.

**`Luke Wroblewski의 반응형 레이아웃 디자인 패턴`**

| 종류                            | 설명                                                                                                           |
| ------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| 유동형(Mostly Fluid)            | 큰 화면에서는 유동형 그리드를 유지하고, 작은 화면에서는 컬럼이 수직으로 쌓이는 패턴. 가장 기본적인 반응형 패턴 |
| 컬럼 드롭(Column Drop)          | 화면이 좁아질수록 다단 컬럼이 아래로 순차적으로 쌓이는 패턴                                                    |
| 레이아웃 시프터(Layout Shifter) | 화면 크기에 따라 요소의 위치 자체가 크게 이동·재배치되는 패턴. 가장 유연하지만 구현이 복잡함                   |
| 미세 조정(Tiny Tweaks)          | 레이아웃은 거의 유지하면서 글자 크기, 여백 등 미세한 요소만 조정하는 패턴. 단일 컬럼 콘텐츠에 적합             |
| 오프 캔버스(Off Canvas)         | 내비게이션이나 보조 콘텐츠를 화면 밖에 숨겨두고, 버튼 클릭 시 슬라이드하여 표시하는 패턴                       |

## UX 적용 및 UI 설계

### UX 디자인

UX는 사용자가 제품 또는 서비스를 사용하면서 상호작용을 통해 얻게 되는 총체적 경험을 의미한다.

UX 디자인은 시각적 요소뿐만 아니라 사용자의 감정, 편의성, 접근성을 고려하여 더 나은 사용자 경험을 제공하는 것을 목표로 한다.

**`UX 사용자 조사 및 분석 방법`**

| 종류                              | 설명                                                                          |
| --------------------------------- | ----------------------------------------------------------------------------- |
| 설문조사                          | 다수의 사용자를 대상으로 정량적 데이터를 수집하는 방법                        |
| 인터뷰                            | 소수의 사용자와 심층 대화를 통해 정성적 데이터를 수집하는 방법                |
| A/B 테스트                        | 두 가지 버전을 각각 사용자 그룹에 노출하여 성과를 비교하는 방법               |
| FGI(Focus Group Interview)        | 소그룹 인터뷰를 통해 집단적 의견과 반응을 수집하는 방법                       |
| Cohort Analysis                   | 특정 기간에 같은 행동을 한 사용자 집단을 그룹화하여 행동 패턴을 분석하는 방법 |
| Heatmap                           | 사용자가 클릭·스크롤한 위치를 시각화하여 행동 패턴을 파악하는 방법            |
| 친화도(Affinity) 다이어그램       | 수집한 데이터를 유사한 주제별로 그룹화하여 패턴과 인사이트를 도출하는 방법    |
| 카드 정렬                         | 사용자가 직접 콘텐츠 카드를 분류하게 하여 정보 구조를 설계하는 방법           |
| 분석도구 활용                     | Google Analytics 등의 도구로 사용자 행동 데이터를 수집·분석하는 방법          |
| 사용성(Usability) 테스트          | 실제 사용자가 제품을 사용하는 과정을 관찰하여 문제점을 발견하는 방법          |
| 사용자 여정지도(User Journey Map) | 사용자가 서비스를 이용하는 전 과정의 경험을 시각화한 도구                     |
| 경험지도(Experience Map)          | 사용자의 여정, 감정, 터치포인트 등을 종합적으로 시각화한 도구                 |

**`경험지도 구성 요소`**

| 요소                       | 설명                                                                |
| -------------------------- | ------------------------------------------------------------------- |
| 페르소나(Persona)          | 서비스를 대표하는 가상의 사용자 유형을 정의한 것                    |
| 타임라인(Timeline)         | 사용자 경험의 흐름을 시간 순서대로 나타낸 축                        |
| 터치 포인트(Touch Point)   | 사용자가 서비스 또는 브랜드와 직접 접촉하는 모든 접점               |
| 채널(Channel)              | 사용자가 서비스와 상호작용하는 매체 (앱, 웹, 오프라인 등)           |
| 감정 상태(Emotional State) | 각 단계에서 사용자가 느끼는 감정. 만족·불만족 등을 곡선 형태로 표현 |
| 페인 포인트(Pain Point)    | 사용자가 경험하는 불편함, 불만, 장애물                              |
| 인사이트(Insight)          | 데이터 분석으로 도출된 핵심 발견과 개선 방향                        |

**`UX 디자인의 접근법`**

| 종류                              | 설명                                                                               |
| --------------------------------- | ---------------------------------------------------------------------------------- |
| 인터렉션(Interaction) 디자인      | 사용자와 시스템 간의 상호작용을 설계하는 접근법. 버튼 반응, 전환 애니메이션 등     |
| 반응형(Responsive) 디자인         | 다양한 화면 크기와 디바이스에 최적화된 레이아웃을 설계하는 접근법                  |
| 사용자 중심(User-centered) 디자인 | 사용자의 요구·행동·목표를 중심으로 제품을 설계하는 접근법                          |
| 비주얼(Visual) 디자인             | 색상, 타이포그래피, 레이아웃 등 시각적 요소를 통해 심미성과 명확성을 높이는 접근법 |
| 윤리적(Ethical) 디자인            | 사용자의 프라이버시·접근성을 고려하고 다크 패턴을 지양하는 책임감 있는 설계 접근법 |

**`UX 제스처(Gesture) 종류`**

| 종류            | 설명                                                                  |
| --------------- | --------------------------------------------------------------------- |
| Press           | 화면을 짧게 누르는 동작. 버튼 클릭, 항목 선택 등에 사용               |
| Long Press      | 화면을 길게 누르는 동작. 컨텍스트 메뉴 열기, 편집 모드 진입 등에 사용 |
| Scroll          | 손가락을 위아래로 미끄러뜨려 콘텐츠를 탐색하는 동작                   |
| Drag            | 요소를 누른 채 이동시키는 동작. 정렬, 위치 변경 등에 사용             |
| Pull to Refresh | 목록 최상단에서 아래로 당겨 콘텐츠를 새로 고침하는 동작               |
| Single Tap      | 화면을 한 번 가볍게 터치하는 동작. 링크 이동, 항목 선택 등에 사용     |
| Double Tap      | 화면을 빠르게 두 번 터치하는 동작. 좋아요, 확대/축소 등에 사용        |
| Pinch           | 두 손가락을 오므리거나 펼치는 동작. 화면 축소/확대에 사용             |

### 디자인 씽킹(Design Thinking)

Design Thinking이란 복잡한 문제를 사용자 중심적으로 해결하는 접근법이다.

Design Thinking을 UX 디자인에 적용하면 모든 단계에서 사용자 중심의 창의적 접근 방식을 적용할 수 있다.

### UX 허니콤 모델(UX Honeycomb Model)

Peter Morville은 UX 허니콤 모델을 통해 UX 디자인을 위한 7요소를 제시했다.

**`Honeycomb model 7요소`**

| 종류        | 설명                                                               |
| ----------- | ------------------------------------------------------------------ |
| 유용성      | 사용자의 필요를 충족하는 기능과 콘텐츠를 제공하는 것               |
| 사용성      | 쉽고 직관적으로 사용할 수 있는 인터페이스를 제공하는 것            |
| 신뢰성      | 사용자가 제품과 브랜드를 신뢰할 수 있도록 하는 것                  |
| 검색 가능성 | 사용자가 필요한 정보를 쉽고 빠르게 찾을 수 있도록 하는 것          |
| 접근성      | 장애 여부나 환경에 관계없이 모든 사용자가 이용할 수 있도록 하는 것 |
| 매력성      | 시각적 요소와 감성적 디자인으로 사용자에게 긍정적 감정을 주는 것   |
| 가치성      | 사용자와 서비스 제공자 모두에게 실질적 가치를 제공하는 것          |

### 페르소나(Persona)

페르소나란 UX 디자인에서 사용자의 특정 특성과 자질을 반영한 가상의 사용자 프로필이며, 가상 캐릭터, 가상 사용자 유형을 나타낸다.

페르소나는 새롭게 분석된 사용자의 행동이나 욕구 변화를 반영하여 지속적으로 업데이트한다.

페르소나는 여러 명일 수도 있다.

**`페르소나 설정 기본 요소`**

| 종류                   | 설명                                                                     |
| ---------------------- | ------------------------------------------------------------------------ |
| 인구 통계 정보         | 이름, 나이, 성별, 직업, 거주지 등 기본적인 인구 통계 데이터              |
| 행동 패턴              | 제품·서비스 사용 방식, 주요 행동 습관, 미디어 소비 패턴 등               |
| 심리적 특성            | 가치관, 동기, 불안 요소, 라이프스타일 등                                 |
| 사용자 목표와 요구사항 | 페르소나가 달성하고자 하는 목표와 서비스에 기대하는 것                   |
| 기타 정보              | 인용구(Quote), 페인 포인트, 기술 숙련도 등 캐릭터를 구체화하는 추가 정보 |

### UI(User Interface) Heuristic

Jakob Nielsen은 UI디자인을 위한 10가지 사용성 휴리스틱을 제시했다.

| 종류                       | 설명                                                                                               |
| -------------------------- | -------------------------------------------------------------------------------------------------- |
| 시스템 상태의 가시성       | 시스템이 현재 어떤 상태인지 사용자에게 항상 피드백을 제공해야 함 (예: 로딩 표시, 저장 완료 메시지) |
| 시스템과 현실 세계의 일치  | 사용자에게 익숙한 실제 세계의 언어·개념·관례를 사용하여 인터페이스를 구성해야 함                   |
| 사용자 제어 및 자유        | 사용자가 실수를 했을 때 쉽게 취소하거나 되돌릴 수 있는 출구(Undo/Redo 등)를 제공해야 함            |
| 일관성 및 표준             | 동일한 기능과 개념에 일관된 단어·디자인을 사용하여 사용자가 혼동하지 않도록 해야 함                |
| 오류 방지                  | 오류 메시지보다 오류 자체가 발생하지 않도록 설계를 통해 사전에 예방해야 함                         |
| 기억보다는 인식            | 사용자가 정보를 기억하지 않아도 되도록, 필요한 옵션·행동을 화면에 명확히 보여줘야 함               |
| 사용의 유연성과 효율성     | 초보자와 숙련자 모두를 위해 단축키, 맞춤 설정 등 효율적인 인터랙션을 지원해야 함                   |
| 미학적이고 미니멀한 디자인 | 불필요한 정보나 요소를 제거하고, 관련성이 높은 정보만 표시하여 인터페이스를 간결하게 유지해야 함   |
| 오류의 인식, 진단 및 복구  | 오류 메시지는 명확한 언어로 문제를 설명하고, 구체적인 해결책을 제시해야 함                         |
| 도움말 및 문서화           | 도움 없이 사용 가능해야 하지만, 필요 시 쉽게 검색할 수 있는 도움말을 제공해야 함                   |

### 기타 UX/UI 용어

| 용어                 | 설명                                                                                                  |
| -------------------- | ----------------------------------------------------------------------------------------------------- |
| 어포던스(Affordance) | UI 요소가 어떻게 사용해야 하는지를 스스로 암시하는 특성 (예: 버튼처럼 보이는 요소는 클릭을 유도)      |
| Agile UX             | 애자일 개발 방법론과 UX 디자인을 결합하여 짧은 스프린트 단위로 반복적으로 디자인·테스트·개선하는 방식 |
| Color Palette        | 브랜드 아이덴티티와 UI 일관성을 위해 디자인에 사용할 색상들을 미리 정의해 놓은 색상 모음              |
| User flow            | 사용자가 특정 목표를 달성하기 위해 거치는 화면과 단계들을 시각화한 흐름도                             |

## 3D 디자인

### 3D 디자인 과정

| 과정           | 설명                                                                                                     |
| -------------- | -------------------------------------------------------------------------------------------------------- |
| Modeling       | 3D 오브젝트의 형태와 구조를 만드는 과정                                                                  |
| Rendering      | 3D 모델에 조명·재질·색상 등을 적용하여 최종 2D 이미지로 변환하는 과정                                    |
| Projection     | 3D 공간의 좌표를 2D 화면 좌표로 변환하는 과정. 원근 투영(Perspective)과 평행 투영(Orthographic)으로 구분 |
| Clipping       | 화면 밖에 위치한 오브젝트나 일부를 제거하여 렌더링 범위를 제한하는 과정                                  |
| Hidden Surface | 시야에서 가려진 면을 제거하여 올바른 깊이감을 표현하는 과정                                              |
| Shading        | 조명과 재질에 따라 각 표면의 밝기와 색상을 계산하는 과정                                                 |
| Mapping        | 3D 모델의 표면에 텍스처나 이미지를 입히는 과정                                                           |

### Mapping 종류

| 종류               | 설명                                                                                         |
| ------------------ | -------------------------------------------------------------------------------------------- |
| texture mapping    | 3D 오브젝트 표면에 2D 이미지(텍스처)를 씌워 색상과 패턴을 표현하는 방식                      |
| normal mapping     | 법선 벡터 정보를 저장한 맵을 사용하여 적은 폴리곤으로도 표면 요철을 사실적으로 표현하는 방식 |
| bump mapping       | 흑백 높이 정보를 사용하여 표면의 미세한 굴곡을 시뮬레이션하는 방식                           |
| reflection mapping | 환경 맵 이미지를 사용하여 오브젝트 표면에 주변 환경의 반사를 표현하는 방식                   |
| opacity mapping    | 알파(투명도) 정보를 사용하여 오브젝트 표면의 투명·불투명 영역을 제어하는 방식                |

### Shading 종류

| 종류            | 설명                                                                                          |
| --------------- | --------------------------------------------------------------------------------------------- |
| Flat shading    | 각 다각형 면 전체에 균일한 단색을 적용하는 방식. 처리 속도가 빠르지만 면 경계가 뚜렷하게 보임 |
| Gouraud shading | 각 정점(Vertex)의 색상을 보간(Interpolation)하여 면 전체에 부드러운 색상 전환을 표현하는 방식 |
| Phong shading   | 각 픽셀 단위로 조명을 계산하여 하이라이트와 반사광을 사실적으로 표현하는 방식                 |
| Metal shading   | 금속 재질의 반사 특성을 시뮬레이션하여 광택과 반사를 사실적으로 표현하는 방식                 |

### 3D 모델링

| 종류             | 설명                                                                                        |
| ---------------- | ------------------------------------------------------------------------------------------- |
| wireframe model  | 오브젝트의 윤곽선과 모서리(Edge)만으로 구성된 뼈대 모델. 형태 파악 및 초기 구조 설계에 사용 |
| vertex model     | 오브젝트를 정점(Vertex) 좌표 데이터만으로 표현하는 모델                                     |
| surface model    | 오브젝트의 외부 표면(면)으로 구성된 모델. 겉면만 정의하고 내부는 비어 있음                  |
| solid model      | 오브젝트의 내부 부피까지 수학적으로 정의한 완전한 3D 모델                                   |
| parametric model | 수학적 매개변수(Parameter)로 형상을 정의하여 매개변수 변경만으로 형태를 조정할 수 있는 모델 |
| fractal model    | 자기 유사성(프랙탈) 수학 공식으로 산, 구름, 파도 등 복잡한 자연물을 표현하는 모델           |
| particle model   | 수많은 작은 입자(Particle)로 화염, 연기, 물 등 유동적인 현상을 표현하는 모델                |

## 웹사이트

### 웹 브라우저 관련 해상도

| 종류   | 픽셀      | 설명                                              |
| ------ | --------- | ------------------------------------------------- |
| HD     | 1280×720  | High Definition. 일반 HD 해상도                   |
| FHD    | 1920×1080 | Full HD. 현재 가장 보편적으로 사용되는 해상도     |
| QHD    | 2560×1440 | Quad HD. FHD의 4배 픽셀 밀도                      |
| 4K UHD | 3840×2160 | Ultra HD. 매우 높은 해상도로 고화질 콘텐츠에 사용 |

### 웹 안전 색상

웹 안전 색상을 활용하면 어떤 환경에서도 동일한 색상이 표시된다.

- 웹으로 보는 컬러는 브라우저 OS, 모니터 종류, 해상도에 따라 다르게 보인다.

웹 안전 색상은 216가지이며, 색상 팔레트 종류를 웹 팔레트(Web Palette)로 선택하여 사용이 가능하다.

### 웹 접근성(Web Accessibility)

웹 접근성이란 모든 사용자가 웹사이트를 쉽게 이용할 수 있도록 보장하는 것을 의미한다.

웹 접근성 기준으로는 W3C에서 제정한 국제 표준 가이드라인 WCAG(Web Content A11y Guidelines)가 있다.

**`WCAG(Web Content A11y Guidelines) 주요 원칙`**

| 종류                        | 설명                                                                                     |
| --------------------------- | ---------------------------------------------------------------------------------------- |
| 인식 가능성(Perceivable)    | 모든 콘텐츠는 사용자가 인식할 수 있어야 함. 이미지에 대체 텍스트(alt) 제공, 자막 제공 등 |
| 운용 가능성(Operable)       | 모든 기능은 키보드로도 조작 가능해야 하며, 충분한 시간과 방법을 제공해야 함              |
| 이해 가능성(Understandable) | 콘텐츠와 UI는 사용자가 이해할 수 있어야 함. 명확한 언어 사용, 오류 메시지 안내 등        |
| 견고성(Robust)              | 다양한 브라우저·보조 기술(스크린 리더 등)에서도 올바르게 해석될 수 있어야 함             |
