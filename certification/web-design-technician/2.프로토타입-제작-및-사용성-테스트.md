## 프로토타입 제작

### 사진 / 이미지 준비

사진이란, 카메라로 촬영한 실물의 장면을 담은 그림을 의미한다.

이미지는 더욱 넓은 의미로서, 그래픽 프로그램으로 생성된 그림, 디지털 방식으로 편집된 사진도 포함된다.

**`웹디자인의 사진 준비 과정`**

| 과정                     | 설명                                                                                           |
| ------------------------ | ---------------------------------------------------------------------------------------------- |
| 1. 목적에 맞는 사진 선택 | 웹페이지의 주제에 맞는 사진, 사용자 경험을 고려하여 긍정적인 경험을 이끌어낼 수 있는 사진 선택 |
| 2. 이미지 해상도 설정    | 편집 SW를 사용하여 해상도를 적절하게 설정                                                      |
| 3. 이미지 편집 및 보정   | 색상을 보정하여 일관된 톤을 유지, 레이아웃에 맞게 사진 크기 조절, 불필요한 배경 및 부분 제거   |
| 4. 이미지 최적화         | 파일 크기를 줄이기 위해 인코딩 포맷 변경                                                       |
| 5. 이미지 업로드 및 배치 | HTML에 삽입                                                                                    |

### 이미지 표현 방식

**`비트맵(Bitmap / Raster) 방식`**

이미지의 색상을 픽셀(Pixel) 단위로 표현하는 방식이다.

이미지를 픽셀로 표현하는 형식을 래스터(Raster)라고 하며, 래스터 형식으로 표현된 이미지를 래스터 이미지라고 한다.

래스터 형식에서 발생하는 계단현상(Jagging)을 보정하기 위해 안티앨리어싱(Anti-aliasing) 기술을 사용한다.

- 모서리 픽셀의 밝기를 조절하여 Jagging을 없앤다.

| 장단점 | 설명                                                                                    |
| ------ | --------------------------------------------------------------------------------------- |
| 장점   | 색상 표현이 풍부하며, 수정에 용이하다                                                   |
| 단점   | 해상도에 의존적이라 크기를 키우면 이미지가 깨짐. 손실 압축 반복 시 품질이 급격히 저하됨 |

**`벡터(Vector) 방식`**

수학적인 계산을 통하여 이미지를 표현하는 방식을 벡터 방식이라고 한다.

| 장단점 | 설명                                                                           |
| ------ | ------------------------------------------------------------------------------ |
| 장점   | 수학적인 계산으로 이미지를 구성하기 때문에 크기 변경에 따른 이미지 손상이 없다 |
| 단점   | 사실적인 이미지(사진 등) 표현에 한계가 있으며, 복잡할수록 렌더링 부하가 증가함 |

**`Rasterizing / Vectorizing`**

| 용어        | 설명                                                                |
| ----------- | ------------------------------------------------------------------- |
| Rasterizing | Vector 방식의 이미지를 Bitmap(Raster) 방식의 이미지로 전환하는 작업 |
| Vectorizing | Bitmap 방식의 이미지를 Vector 방식의 이미지로 전환하는 작업         |

### 색상 체계

색상 체계(Color model)란, 색상을 표현하는 방법을 정의하는 시스템이다.

- 색상 체계는 **색상 모드** 라고도 불린다.

대표적인 색상 체계로 RGB와 CMYK 모드가 있다.

| 색상 모드 | 설명                                                                                                                                                            |
| --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RGB 컬러  | 혼합할수록 색상이 밝아지는 **가산혼합** 방식<br/>모니터, 디바이스 등 디지털 디스플레이, 웹에서 사용됨<br/>색상 값은 0~255로 표현                                |
| CMYK 컬러 | 혼합할수록 색상이 어두워지는 **감산혼합** 방식<br/>CMY색상을 혼합했을 때, 완벽한 검정색이 나오지 않으므로 K(Black)을 추가<br/>각 원색의 값은 0~100%로 표현<br/> |

- 모니터와 출력물의 인쇄 색상이 다른 이유는 디스플레이 컬러 모드(RGB)와 인쇄 컬러 모드(CMYK)가 다르기 때문이다.

**`4도 분판 인쇄`**

인쇄기 출력은 각각의 색상을 하나씩 추가하는 방식을 사용한다. 이를 위해서 CMYK 4개의 필름(Negative film)으로 분해를 하여 색상을 입힌다. 이를 4도 분판 인쇄라고 한다.

현재는 디지털 인쇄가 보편화 되면서, CTP(Computer to Plate) 방식을 사용할 경우 필름 없이 디지털 데이터에서 바로 인쇄판을 제작한다. 분판은 디지털 방식으로 처리된다.

**`기타 색상 체계`**

| 종류                     | 설명                                                                                                                                    |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| 인덱스 컬러(Index Color) | 래스터 이미지의 크기를 줄이기 위해, 자주 쓰이는 256색상을 선별하여 적용한 컬러 모드                                                     |
| 그레이스케일(Grayscale)  | 이미지의 컬러 정보를 없애고, 픽셀이 가질 수 있는 색상을 회색 음영계의 256개(R=G=B)로 한정                                               |
| 듀오톤(Duotone)          | Grayscale 모드 이미지에서 한 두가지의 색상을 추가하여 표현하는 방식                                                                     |
| 비트맵 컬러(Bitmap)      | 검정과 흰색 픽셀 두 가지 색상으로 표현하는 방식                                                                                         |
| Lab 컬러                 | 정확한 색상 측정을 위해 국제 표준으로 제안한 색상 체계<br/> `명도(Lightness) / a(Green / Red) / b(Blue / Yellow)` 세 가지 성분으로 구분 |
| HSB 컬러                 | `색조(Hue) / 채도(Saturation) / 명도(Brightness)` 로 구분하여 표현                                                                      |

### 해상도(Resolution)

**`해상도 종류`**

| 종류          | 설명                                                                                                                                           |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| 이미지 해상도 | **비트맵 방식** 이미지에서 사용되는 이미지 표현에 대한 정밀도<br/>일반적으로 단위 크기당 픽셀의 수로 측정되며, ppi(pixel per inch) 단위를 사용 |
| 화면 해상도   | 디스플레이 화면에 배치된 픽셀에 대한 척도, 모니터 해상도라고도 불린다                                                                          |
| 출력 해상도   | 프린터 등 출력 장치의 정밀도. 출력 해상도는 lpi(line per inch) 단위를 사용하며, 잉크 인쇄시 dpi(dot per inch) 단위를 사용                      |

- lpi는 1인치당 몇 개의 망점(halftone dot)이 들어가는지 나타내므로, 망점의 크기는 lpi에 따라 결정된다.

- CMYK 출력에서 각 색상 판을 서로 다른 각도로 인쇄한다. 이는 모아레(Moiré) 현상을 방지하고 망점들이 규칙적으로 배치되어 자연스러운 색상 혼합이 이루어지도록 하기 위함이다. 이때 형성되는 꽃 모양의 규칙적인 패턴을 **로제트 패턴(rosette pattern)** 이라고 한다.

**`비트 심도(Bit Depth)`**

하나의 픽셀이 색상을 표현하기 위해 사용하는 비트의 수를 뜻한다.

RGB컬러 모드의 경우, R·G·B 각 채널에 8bit씩 할당되어 하나의 픽셀당 $8\text{bit} \times 3\text{(RGB)} = 24\text{bit}$ 가 사용된다. 이 경우 RGB 모드는 24bit의 비트 심도를 가진다고 표현한다. (채널당 $2^8 = 256$ 가지 표현 가능)

**`트루컬러(True Color)`**

트루컬러는 RGB 각 채널에 8bit씩 사용하여 $2^{24} = 16,777,216$ (약 1,677만) 가지 색을 표현하는 컬러 모드로, 인간의 눈이 구분할 수 있는 색상 수에 근접한다.

색 심도(color depth) 기준으로 일반적으로 24bit 이상의 비트 심도를 가진 컬러 모드가 트루컬러 범주에 포함된다.

### 이미지 파일 포맷

**`GIF(Graphic Interchange Format)`**

저속모뎀 통신을 목적으로 개발한 파일 포맷.

무손실 압축 포맷이다.

8bit를 지원하여 최대 256색상을 지원

무손실 압축기법인 **LZW(Lempel-Ziv-Welch)** 를 이용하기 때문에 같은 색상이 수평으로 배치될 경우 높은 압축률을 보인다.

**`PNG(Portable Network Graphics)`**

GIF포맷을 대체하기 위해 개발되었다.

GIF와 비슷하지만 애니메이션을 만들 수 없다.

풀컬러를 사용하여 섬세한 이미지를 나타낼 수 있지만 용량이 크다.

알파채널을 지원하여, 투명도를 사용할 수 있다.

**[`JPEG(Joint Photographic Experts Group)`](../../FE/concepts/JPEG.md#joint-photographic-experts-group)**

**`WebP`**

웹에서 이미지 로딩시간을 단축하기 위해 Google에서 개발한 포맷.

무손실 및 손실 압축을 모두 지원한다.

JPEG와 마찬가지로 XMP(Extensible Metadata Platform) / EXIF(Exchangeable Image File Format) 메타데이터를 포함할 수 있다.

**`기타 이미지 포맷`**

| 종류 | 설명                                               | 특징                                                                   |
| ---- | -------------------------------------------------- | ---------------------------------------------------------------------- |
| bmp  | Windows 기본 비트맵 이미지 포맷                    | 무압축으로 파일 크기 매우 큼. 색 손실 없음. Windows 환경에서 기본 사용 |
| tiff | Tagged Image File Format. 인쇄·출판 업계 표준 포맷 | 무손실 압축 지원. 다양한 색상 깊이(8·16·32bit) 지원. 파일 크기 큼      |
| ai   | Adobe Illustrator 전용 벡터 포맷                   | 벡터 그래픽·텍스트·색상 정보 포함. 인쇄·로고·일러스트에 사용           |
| eps  | Encapsulated PostScript. PostScript 기반 포맷      | 벡터 및 비트맵 모두 포함 가능. 인쇄·출판·다양한 프로그램 간 호환       |
| pdf  | Adobe의 Portable Document Format                   | 텍스트·이미지·벡터를 하나의 파일에 포함. 인쇄·배포의 표준 포맷         |
| psd  | Adobe Photoshop Document. Photoshop 전용 포맷      | 레이어·마스크·채널 등 편집 정보 보존. 파일 크기 큼. 작업 파일로 사용   |
| tga  | Truevision TARGA. 알파채널 지원 포맷               | 고색상 심도(16·24·32bit). 무손실. 게임 그래픽·3D 렌더링에 사용         |
| pict | Apple Macintosh 전용 이미지 포맷                   | 비트맵·벡터 모두 지원. 현재는 거의 사용하지 않음                       |
| raw  | 디지털 카메라의 미처리 이미지 데이터               | 카메라 제조사별 고유 포맷(CR2·NEF·ARW 등). 후보정용 원본 파일          |

## 영상 / 아이콘 / 서체

### 영상

영상은 정지 이미지와 움직이는 동영상을 포괄하는 넓은 의미의 개념이다.

**`영상 파일 포맷`**

| 종류                 | 설명                                                 |
| -------------------- | ---------------------------------------------------- |
| mp4                  | 높은 압축률 / 좋은 품질 제공                         |
| webm                 | HTML5 표준을 지원하는 웹용 포맷                      |
| ogv                  | 오픈소스 포맷                                        |

| srt(SubRip Subtitle) | 비디오 자막 포맷                                     |

### 아이콘

아이콘은 시각적 상징 요소로, 콘텐츠를 시각적으로 식별하고 표현하기 위해 사용된다.

특정 기능을 시각적으로 표현하여 사용성을 높이는 목적으로 주로 사용된다.

**`아이콘 파일 포맷`**

| 종류 | 설명                                                       |
| ---- | ---------------------------------------------------------- |
| svg  | 벡터 그래픽 포맷                                           |
| ico  | MS 윈도우에서 아이콘에 쓰이는 포맷, 주로 웹 favicon에 사용 |

### 서체

서체(Typeface)란 글자의 디자인 그 자체를 의미한다.

**`서체의 종류`**

| 종류       | 설명                              |
| ---------- | --------------------------------- |
| Serif      | 글자의 끝 부분에 돌기가 있는 서체 |
| Sans-serif | 글자의 끝 부분에 돌기가 없는 서체 |
| Script     | 필기체처럼 손으로 쓴 것 같은 서체 |

**`폰트 파일 포맷`**

| 종류               | 설명                                                      |
| ------------------ | --------------------------------------------------------- |
| ttf(TrueType Font) | 벡터 기반 글꼴 포맷                                       |
| otf(OpenType Font) | 고해상도 출력에 사용되는 포맷, ttf와 마찬가지로 벡터 기반 |

### 타이포그래피(Typography)

타이포그래피란, 가독성 및 콘텐츠 컨셉에 맞추어 텍스트를 디자인하고, 배치하는 기술 및 분야를 의미한다.

Typography는 그리스어인 Typos(두드려서 만든 부호/표시)와 Graphia(쓰다/그리다)에서 유래된 조어이다.

**`레터링 디자인(Lettering design)`**

개별 글자를 독창적인 스타일로 디자인하는 분야.

넓은 의미로 글자 디자인을 의미하며, 좁은 의미로 손으로 제작한 문자를 의미한다.

**`가독성(Legibility)`**

| 용어        | 차이점                                                                                                 |
| ----------- | ------------------------------------------------------------------------------------------------------ |
| Readability | 텍스트 전체를 얼마나 쉽게 읽고 이해할 수 있는지에 대한 척도. 행간·자간·줄 길이 등 레이아웃 전반과 관련 |
| Legibility  | 개별 문자를 얼마나 쉽게 식별·구분할 수 있는지에 대한 척도. 서체 자체의 형태적 명확성과 관련            |

**`타이포그래피 비례 요소`**

| 요소            | 설명                                                                       |
| --------------- | -------------------------------------------------------------------------- |
| 장체(Condensed) | 글자의 가로 폭을 좁힌 서체. 세로 방향이 강조되어 날카롭고 긴장감 있는 느낌 |
| 평체(Expended)  | 글자의 가로 폭을 넓힌 서체. 안정적이고 넓은 느낌                           |

### 애니메이션(Animation)

애니메이션의 어원은 라틴어 **Animatus** 에서 유래되었으며, 이는 생명을 불어넣다는 뜻이다.

**`애니메이션 방식`**

| 방식               | 설명                                                                                     |
| ------------------ | ---------------------------------------------------------------------------------------- |
| 프레임(Frame)      | 각 프레임을 연속적으로 보여주는 방식                                                     |
| 키프레임(Keyframe) | 애니메이션의 시작과 끝을 지정하여 중간 프레임을 수학적 보간법으로 계산하여 표현하는 방식 |

- 중간 프레임을 자동으로 생성하는 과정을 **트위닝(Tweening)** 이라고 한다.

**`애니메이션 시각효과`**

| 종류                      | 설명                                                                                         |
| ------------------------- | -------------------------------------------------------------------------------------------- |
| 모핑(Morphing)            | 서로 다른 이미지 사이 프레임을 컴퓨터 그래픽스의 특수 보간법을 통해 점진적으로 표현하는 기법 |
| 크로마키(Chroma-key)      | 특정 색상(주로 초록·파란 배경)을 제거하고 다른 배경 이미지를 합성하는 그래픽스 기술          |
| 모션 캡처(Motion capture) | 실제 모델의 움직임을 추적한 데이터를 캐릭터에 적용하는 기술                                  |
| 콤마(Comma) 촬영          | 영상을 프레임 단위로 나누어 촬영하는 기법                                                    |

**`애니메이션 종류`**

| 종류                             | 설명                                                                                           |
| -------------------------------- | ---------------------------------------------------------------------------------------------- |
| 셀(Cel, celluloid) 애니메이션    | 배경은 그대로 두고, 캐릭터만 움직이는 기법                                                     |
| 스톱모션(Stop-Motion) 애니메이션 | 한 프레임씩 촬영 후, 각 프레임을 연결하여 영사                                                 |
| 고모션(Go-Motion) 애니메이션     | Stop-Motion의 특수 기법으로, 촬영 대상을 기계적으로 움직이게 하여 motion blur 효과를 주는 기법 |
| 컷아웃(Cut-out) 애니메이션       | 특정한 형태(캐릭터 등)의 종이들을 화면에 붙이거나 떼면서(또는 관절을 움직이며) 촬영            |
| 투광(Back-Light) 애니메이션      | Light table에 특정 물체를 놓아서 그 틈으로 나오는 빛을 촬영                                    |
| 로토스코핑(Rotoscoping)          | 실사 영상 위에 프레임마다 그림을 그려 올려 자연스러운 움직임을 표현하는 기법                   |
| 플립북(Flip-book) 애니메이션     | 책 또는 노트에 변해가는 동작을 묘사한 후, 일정한 속도로 페이지를 넘기는 방법                   |

### 아이디어 시각화 방법

| 방법         | 설명                                                              |

| 2차원적 방법 | 그래프, 차트와 같이 2차원적 속성을 이용하여 표현                  |

## 프로토타입 제작과 종류

### 프로토타이핑(Prototyping)

초기 시제품 모델(prototype)을 제작하는 과정을 프로토타이핑이라고 한다.

**`피델리티(Fidelity, 충실도)`**

프로토타입 또는 모델의 묘사나 표현의 정도를 피델리티라고 한다.

피델리티 수준은 프로젝트 단계, 목표, 테스트 목적 등에 따라 결정된다.

| Fidelity 수준 | 설명                                              |
| ------------- | ------------------------------------------------- |
| Lo-Fi         | 자원 절약을 위해 기본적인 형태와 개념 정도만 포함 |
| Mid-Fi        | Lo-Fi에서 간단한 인터랙션을 포함                  |
| High-Fi       | 실제 제품과 유사한 수준                           |

### 프로토타입 종류

| 종류                | 설명                                          |
| ------------------- | --------------------------------------------- |
| 종이 프로토타입     | 종이와 펜만을 사용하여 간단히 스케치하여 표현 |
| 디지털 프로토타입   | Figma와 같은 SW를 사용하여 제작               |
| 코드화된 프로토타입 | 실제 코드로 구현하여 최종 제품과 비슷         |
| Low-Fi 프로토타입   | Low-Fi수준의 프로토타입                       |
| High-Fi 프로토타입  | 최종제품과 비슷한 프로토타입                  |

## 사용자 조사 및 분석

### 사용자 조사 기획

사용자 조사 기획은 사용자의 요구, 행동, 태도, 경험 등을 이해하기 위해 체계적으로 조사하고 계획하는 실행 과정이다.

### 사용자 행동 분석

사용자 행동 분석을 통해 사용자가 제품이나 서비스와 상호작용하는 동안 수행하는 행동을 파악하고 이해한다.

**`사용자 행동 분석 요소`**

| 요소                          | 설명                                                          |
| ----------------------------- | ------------------------------------------------------------- |
| Page View                     | 웹 페이지를 열어본 수                                         |
| 클릭률(Click Through Rate)    | 노출 대비 클릭한 링크나 버튼의 비율                           |
| 페이지 체류시간(Time on Page) | 특정 페이지에 머문 시간                                       |
| 이탈률(Bounce Rate)           | 첫 페이지에 방문한 후 다른 페이지로 이동하지 않고 이탈한 비율 |
| 전환율(Conversion Rate)       | 목표 행동(구매, 가입 등)을 한 비율                            |

**`사용자 행동 분석 방법`**

| 종류                           | 설명                                                                   |
| ------------------------------ | ---------------------------------------------------------------------- |
| 웹 로그(Web log) 분석          | 사용자의 상호작용 데이터(IP, Timestamp, event type등)를 서버에 기록    |
| 클릭스트림(Clickstream) 분석   | 클릭을 통한 이동 경로 분석                                             |
| 사용자 여정(User Journey) 분석 | 제품을 처음 접한 순간부터 끝낼 때까지의 모든 과정을 추적 및 분석       |
| 히트맵(Heatmap) 분석           | 히트맵을 활용하여 시각적으로 분석                                      |
| A/B 테스트                     | 두 가지(A, B) 다른 버전을 제공하여, 각 버전에 대한 행동 및 반응을 측정 |
| 코호트(Cohort) 분석            | 특정 시점에서 동일한 특성을 가진 사용자 그룹(Cohort)를 추적하고 분석   |

### 사용자 태도 분석

사용자 태도 분석을 통해 제품이나 서비스에 대한 사용자의 **주관적인** 경험을 파악하여, 행동 분석으로 파악하기 어려운 숨겨진 문제점을 찾아낸다.

태도 분석은 데이터 분석적인 측면보다, 사용자의 내면적인 생각과 감정을 이해하는 것에 중점을 둔다.

태도 분석은 정성적인 측면이 강하고, 행동 분석은 정량적은 측면이 강하다.

### 사용성 테스트(Usability Test)

사용성 테스트는 웹사이트가 얼마나 효율적으로 사용될 수 있는지 평가하는 과정을 말한다.

프로토타입을 기반으로 실제 사용자에게 테스트를 실시하여 피드백을 수집한다.

사용성 개선을 위해 UX를 중심으로 **형성적 사용적합성 평가(Formative Usability Testing)** 를 실시한다.

**`사용성 테스트 준비사항`**

| 종류        | 설명                                                                                                                                      |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| 테스트 항목 | 방문 목적에 따라 평가 기준을 결정<br/>사이트 위치의 정확성, 이동의 용이성, 레이아웃, 메뉴 배치, 검색 기능, 반복 사용성 등을 기준으로 평가 |
| 테스트 방법 | 질문법: 조사법, 설문법<br/>감정법: 발견적 평가, 속성 검사, 안구 추적, 수행 측정 등                                                        |
| 목표 설정   | 테스트를 통해 확인하고자 하는 목표를 명확히 정의<br/>사이트 역할, 방문 목적, 이용하는 서비스, 기능의 사용 용이성 등의 목표를 설정         |

**`형성적 사용적합성 평가 항목`**

| 항목          | 설명                                                              |
| ------------- | ----------------------------------------------------------------- |
| 정보의 효율성 | 필요한 정보를 얼마나 빨리 찾을 수 있는지 측정                     |
| 작업의 효율성 | 필요한 작업을 수행하는 데 걸리는 시간과 노력을 평가               |
| 만족도        | 웹사이트를 얼마나 만족스럽게 사용하는지 평가                      |
| 직관성        | 내비게이션, 인터페이스가 얼마나 직관적인지 평가                   |
| 접근성        | 다양한 사용자들에 대해 얼마나 쉽게 접근하고 사용할 수 있는지 평가 |
| 응답성        | 다양한 장치에 대해서 기능을 잘 수행하는지 평가                    |

**`발견적 평가(Heuristic Evaluation)`**

Jakob Nielsen이 주장한 사용성 방법으로, 전문가들이 웹사이트나 인터페이스를 검사하여 사용성 문제를 발견하는 방법이다.

사용성을 평가하기 위한 10가지 휴리스틱 원리를 제안한다.

| 종류                          | 설명                                                                      |
| ----------------------------- | ------------------------------------------------------------------------- |
| 시스템 상태의 가시성 유지     | 시스템의 상태를 사용자에게 알림                                           |
| 실제 세상과 시스템의 일치     | 사용자에게 익숙한 단어 및 구문 사용                                       |
| 사용자 제어 및 자유           | redo 및 undo 지원                                                         |
| 일관성과 표준                 | 다른 단어, 상황, 동작이 동일한 의미를 가지지 않도록 명확히 구분           |
| 오류 예방                     | 예외 발생 가능성 제거, 또는 오류 발생시 사용자에게 알림 및 확인 옵션 제공 |
| 회상 보다는 인식              | 별도 교육 없이 기능을 쉽게 인식하여 사용할 수 있도록 함                   |
| 유연성과 효율성               | 초보 사용자 및 숙련 사용자 모두를 위한 인터페이스 제공                    |
| 심미적이고 최소의 디자인 제공 | 관련없는 정보를 포함하지 않도록 함                                        |
| 오류로부터 회복 지원          | 사용자가 오류를 인식, 진단 및 복구할 수 있도록 지원                       |
| 도움말 및 문서 제공           | 레퍼런스 및 docs 제공                                                     |
