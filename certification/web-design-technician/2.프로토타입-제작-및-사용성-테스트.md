## 프로토타입 제작

### 사진 / 이미지 준비

사진이란, 카메라로 촬영한 실물의 장면을 담은 그림을 의미한다.

이미지는 더욱 넓은 의미로서, 그래픽 프로그램으로 생성된 그림, 디지털 방식으로 편집된 사진도 포함된다.

**`웹디자인의 사진 준비 과정`**

| 과정                     | 설명                                                                                           |
| ------------------------ | ---------------------------------------------------------------------------------------------- |
| 1. 목적에 맞는 사진 선택 | 웹페이지의 주제에 맞는 사진, 사용자 경험을 고려하여 긍정적인 경험을 이끌어낼 수 있는 사진 선택 |
| 2. 이미지 해상도 설정    | 편집 SW를 사용하여 해상도를 적절하게 설정                                                      |
| 3. 이미지 편집 및 보정   | 색상을 보정하여 일관된 톤을 유지, 레이아웃에 맞게 사진 크기 조절, 불필요한 배경 및 부분 제거   |
| 4. 이미지 최적화         | 파일 크기를 줄이기 위해 인코딩 포맷 변경                                                       |
| 5. 이미지 업로드 및 배치 | HTML에 삽입                                                                                    |

### 이미지 표현 방식

**`비트맵(Bitmap / Raster) 방식`**

이미지의 색상을 픽셀(Pixel) 단위로 표현하는 방식이다.

이미지를 픽셀로 표현하는 형식을 래스터(Raster)라고 하며, 래스터 형식으로 표현된 이미지를 래스터 이미지라고 한다.

래스터 형식에서 발생하는 계단현상(Jagging)을 보정하기 위해 안티앨리어싱(Anti-aliasing) 기술을 사용한다.

- 모서리 픽셀의 밝기를 조절하여 Jagging을 없앤다.

| 장단점 | 설명                                                                                    |
| ------ | --------------------------------------------------------------------------------------- |
| 장점   | 색상 표현이 풍부하며, 수정에 용이하다                                                   |
| 단점   | 해상도에 의존적이라 크기를 키우면 이미지가 깨짐. 손실 압축 반복 시 품질이 급격히 저하됨 |

**`벡터(Vector) 방식`**

수학적인 계산을 통하여 이미지를 표현하는 방식을 벡터 방식이라고 한다.

| 장단점 | 설명                                                                           |
| ------ | ------------------------------------------------------------------------------ |
| 장점   | 수학적인 계산으로 이미지를 구성하기 때문에 크기 변경에 따른 이미지 손상이 없다 |
| 단점   | 사실적인 이미지(사진 등) 표현에 한계가 있으며, 복잡할수록 렌더링 부하가 증가함 |

**`Rasterizing / Vectorizing`**

| 용어        | 설명                                                                |
| ----------- | ------------------------------------------------------------------- |
| Rasterizing | Vector 방식의 이미지를 Bitmap(Raster) 방식의 이미지로 전환하는 작업 |
| Vectorizing | Bitmap 방식의 이미지를 Vector 방식의 이미지로 전환하는 작업         |

### 색상 체계

색상 체계(Color model)란, 색상을 표현하는 방법을 정의하는 시스템이다.

- 색상 체계는 **색상 모드** 라고도 불린다.

대표적인 색상 체계로 RGB와 CMYK 모드가 있다.

| 색상 모드 | 설명                                                                                                                                                            |
| --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RGB 컬러  | 혼합할수록 색상이 밝아지는 **가산혼합** 방식<br/>모니터, 디바이스 등 디지털 디스플레이, 웹에서 사용됨<br/>색상 값은 0~255로 표현                                |
| CMYK 컬러 | 혼합할수록 색상이 어두워지는 **감산혼합** 방식<br/>CMY색상을 혼합했을 때, 완벽한 검정색이 나오지 않으므로 K(Black)을 추가<br/>각 원색의 값은 0~100%로 표현<br/> |

- 모니터와 출력물의 인쇄 색상이 다른 이유는 디스플레이 컬러 모드(RGB)와 인쇄 컬러 모드(CMYK)가 다르기 때문이다.

**`4도 분판 인쇄`**

인쇄기 출력은 각각의 색상을 하나씩 추가하는 방식을 사용한다. 이를 위해서 CMYK 4개의 필름(Negative film)으로 분해를 하여 색상을 입힌다. 이를 4도 분판 인쇄라고 한다.

현재는 디지털 인쇄가 보편화 되면서, CTP(Computer to Plate) 방식을 사용할 경우 필름 없이 디지털 데이터에서 바로 인쇄판을 제작한다. 분판은 디지털 방식으로 처리된다.

**`기타 색상 체계`**

| 종류                     | 설명                                                                                                                                    |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| 인덱스 컬러(Index Color) | 래스터 이미지의 크기를 줄이기 위해, 자주 쓰이는 256색상을 선별하여 적용한 컬러 모드                                                     |
| 그레이스케일(Grayscale)  | 이미지의 컬러 정보를 없애고, 픽셀이 가질 수 있는 색상을 회색 음영계의 256개(R=G=B)로 한정                                               |
| 듀오톤(Duotone)          | Grayscale 모드 이미지에서 한 두가지의 색상을 추가하여 표현하는 방식                                                                     |
| 비트맵 컬러(Bitmap)      | 검정과 흰색 픽셀 두 가지 색상으로 표현하는 방식                                                                                         |
| Lab 컬러                 | 정확한 색상 측정을 위해 국제 표준으로 제안한 색상 체계<br/> `명도(Lightness) / a(Green / Red) / b(Blue / Yellow)` 세 가지 성분으로 구분 |
| HSB 컬러                 | `색조(Hue) / 채도(Saturation) / 명도(Brightness)` 로 구분하여 표현                                                                      |

### 해상도(Resolution)

**`해상도 종류`**

| 종류          | 설명                                                                                                                                           |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| 이미지 해상도 | **비트맵 방식** 이미지에서 사용되는 이미지 표현에 대한 정밀도<br/>일반적으로 단위 크기당 픽셀의 수로 측정되며, ppi(pixel per inch) 단위를 사용 |
| 화면 해상도   | 디스플레이 화면에 배치된 픽셀에 대한 척도, 모니터 해상도라고도 불린다                                                                          |
| 출력 해상도   | 프린터 등 출력 장치의 정밀도. 출력 해상도는 lpi(line per inch) 단위를 사용하며, 잉크 인쇄시 dpi(dot per inch) 단위를 사용                      |

- lpi는 1인치당 몇 개의 망점(halftone dot)이 들어가는지 나타내므로, 망점의 크기는 lpi에 따라 결정된다.

- CMYK 출력에서 각 색상 판을 서로 다른 각도로 인쇄한다. 이는 모아레(Moiré) 현상을 방지하고 망점들이 규칙적으로 배치되어 자연스러운 색상 혼합이 이루어지도록 하기 위함이다. 이때 형성되는 꽃 모양의 규칙적인 패턴을 **로제트 패턴(rosette pattern)** 이라고 한다.

**`비트 심도(Bit Depth)`**

하나의 픽셀이 색상을 표현하기 위해 사용하는 비트의 수를 뜻한다.

RGB컬러 모드의 경우, R·G·B 각 채널에 8bit씩 할당되어 하나의 픽셀당 $8\text{bit} \times 3\text{(RGB)} = 24\text{bit}$ 가 사용된다. 이 경우 RGB 모드는 24bit의 비트 심도를 가진다고 표현한다. (채널당 $2^8 = 256$ 가지 표현 가능)

**`트루컬러(True Color)`**

트루컬러는 RGB 각 채널에 8bit씩 사용하여 $2^{24} = 16,777,216$ (약 1,677만) 가지 색을 표현하는 컬러 모드로, 인간의 눈이 구분할 수 있는 색상 수에 근접한다.

색 심도(color depth) 기준으로 일반적으로 24bit 이상의 비트 심도를 가진 컬러 모드가 트루컬러 범주에 포함된다.

### 이미지 파일 포맷

**`GIF(Graphic Interchange Format)`**

저속모뎀 통신을 목적으로 개발한 파일 포맷.

무손실 압축 포맷이다.

8bit를 지원하여 최대 256색상을 지원

무손실 압축기법인 **LZW(Lempel-Ziv-Welch)** 를 이용하기 때문에 같은 색상이 수평으로 배치될 경우 높은 압축률을 보인다.

**`PNG(Portable Network Graphics)`**

GIF포맷을 대체하기 위해 개발되었다.

GIF와 비슷하지만 애니메이션을 만들 수 없다.

풀컬러를 사용하여 섬세한 이미지를 나타낼 수 있지만 용량이 크다.

알파채널을 지원하여, 투명도를 사용할 수 있다.

**[`JPEG(Joint Photographic Experts Group)`](../../FE/concepts/JPEG.md#joint-photographic-experts-group)**

**`WebP`**

웹에서 이미지 로딩시간을 단축하기 위해 Google에서 개발한 포맷.

무손실 및 손실 압축을 모두 지원한다.

JPEG와 마찬가지로 XMP(Extensible Metadata Platform) / EXIF(Exchangeable Image File Format) 메타데이터를 포함할 수 있다.

**`기타 이미지 포맷`**

| 종류 | 설명                                                                |
| ---- | ------------------------------------------------------------------- |
| bmp  | 비트맵 이미지 포맷                                                  |
| tiff | 인쇄 목적 포맷, 호환성이 뛰어남                                     |
| ai   | 비트맵 / 벡터 이미지를 동시에 저장, 일러스트레이터의 기본 저장 포맷 |
| eps  | 벡터 기반 포맷. 인쇄 출판에 주로 사용되며 CMYK 분판 출력에 적합     |
| pdf  | 디지털 출판을 목적으로 Adobe에서 개발된 포맷                        |
| psd  | 포토샵 원본 파일 포맷                                               |

## 영상 / 아이콘 / 서체

### 영상

영상은 정지 이미지와 움직이는 동영상을 포괄하는 넓은 의미의 개념이다.

**`영상 파일 포맷`**

| 종류                 | 설명                                                 |
| -------------------- | ---------------------------------------------------- |
| mp4                  | 높은 압축률 / 좋은 품질 제공                         |
| webm                 | HTML5 표준을 지원하는 웹용 포맷                      |
| ogv                  | 오픈소스 포맷                                        |
| mov                  | Apple의 QuickTime Player 형식 포맷, 고품질 및 고용량 |
| avi                  | 다양한 코덱을 지원하는 포맷                          |
| swf                  | Adobe에서 개발된 플래시 프로그램 확장자              |
| srt(SubRip Subtitle) | 비디오 자막 포맷                                     |

### 아이콘

아이콘은 시각적 상징 요소로, 콘텐츠를 시각적으로 식별하고 표현하기 위해 사용된다.

특정 기능을 시각적으로 표현하여 사용성을 높이는 목적으로 주로 사용된다.

**`아이콘 파일 포맷`**

| 종류 | 설명                                                       |
| ---- | ---------------------------------------------------------- |
| svg  | 벡터 그래픽 포맷                                           |
| ico  | MS 윈도우에서 아이콘에 쓰이는 포맷, 주로 웹 favicon에 사용 |

### 서체

서체(Typeface)란 글자의 디자인 그 자체를 의미한다.

**`서체의 종류`**

| 종류       | 설명                              |
| ---------- | --------------------------------- |
| Serif      | 글자의 끝 부분에 돌기가 있는 서체 |
| Sans-serif | 글자의 끝 부분에 돌기가 없는 서체 |
| Script     | 필기체처럼 손으로 쓴 것 같은 서체 |

**`폰트 파일 포맷`**

| 종류               | 설명                                                      |
| ------------------ | --------------------------------------------------------- |
| ttf(TrueType Font) | 벡터 기반 글꼴 포맷                                       |
| otf(OpenType Font) | 고해상도 출력에 사용되는 포맷, ttf와 마찬가지로 벡터 기반 |

### 타이포그래피(Typography)

타이포그래피란, 가독성 및 콘텐츠 컨셉에 맞추어 텍스트를 디자인하고, 배치하는 기술 및 분야를 의미한다.

**`레터링 디자인(Lettering design)`**

개별 글자를 독창적인 스타일로 디자인하는 분야.

넓은 의미로 글자 디자인을 의미하며, 좁은 의미로 손으로 제작한 문자를 의미한다.

### 애니메이션(Animation)

애니메이션의 어원은 라틴어 **Animatus** 에서 유래되었으며, 이는 생명을 불어넣다는 뜻이다.

**`애니메이션 방식`**

| 방식               | 설명                                                                                      |
| ------------------ | ----------------------------------------------------------------------------------------- |
| 프레임(Frame)      | 각 프레임을 연속적으로 보여주는 방식                                                      |
| 키프레임(Keyframe) | 애니메이션의 시작과 끝을 지정하여 중간 프레임s을 수학적 보간법으로 계산하여 표현하는 방식 |

- 중간 프레임을 자동으로 생성하는 과정을 **트위닝(Tweening)** 이라고 한다.

**`애니메이션 시각효과`**

| 종류                      | 설명                                                                                         |
| ------------------------- | -------------------------------------------------------------------------------------------- |
| 모핑(Morphing)            | 서로 다른 이미지 사이 프레임을 컴퓨터 그래픽스의 특수 보간법을 통해 점진적으로 표현하는 기법 |
| 크로마키(Chroma-key)      | 특정 색상(주로 초록·파란 배경)을 제거하고 다른 배경 이미지를 합성하는 그래픽스 기술          |
| 모션 캡처(Motion capture) | 실제 모델의 움직임을 추적한 데이터를 캐릭터에 적용하는 기술                                  |
| 콤마(Comma) 촬영          | 영상을 프레임 단위로 나누어 촬영하는 기법                                                    |

## 아이디어 시각화

### 아이디어 시각화 방법

| 방법         | 설명                                                              |
| ------------ | ----------------------------------------------------------------- |
| 1차원적 방법 | 정보를 일렬로 배열하여 표현, 텍스트나 음성 출력, 점자를 통한 표현 |
| 2차원적 방법 | 그래프, 차트와 같이 2차원적 속성을 이용하여 표현                  |
| 3차원적 방법 | 3D 모델을 통한 표현                                               |

**`에스키스(Esquisse)`**

시각화 과정 초기에 대략적인 내용을 간단히 스케치하는 과정.

### 데이터 시각화 방법

| 방법                         | 설명                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| 히트맵(Heatmap)              | 데이터 값의 크기 및 빈도를 색상으로 표현(녹색 &rarr; 빨간색) |
| 트리맵(Treemap)              | 계층구조 데이터를 직사각형 타일로 표현                       |
| 쌍곡선 트리(Hyperbolic tree) | 계층구조 데이터를 쌍곡면 상에 시각적으로 표현                |
