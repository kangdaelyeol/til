## 기초데이터와 레퍼런스 데이터 수집

### 프로토타입

프로토타입은 **시제품** 이라고도 하며, 최종 제품과 실제 사용자와 상호작용을 위해 구현한 시뮬레이션 모델이다.

프로토타입을 제작하기 위해서는 기초데이터 수집과 레퍼런스 조사가 필요하다.

### 기초데이터

기초데이터는 사용자요구, 시장 조사, 기술적 요구사항 등을 포함하는 정보로 프로젝트의 기초를 형성하는 모든 데이터를 말한다.

**`기초데이터 수집 방법`**

| 이름     | 방법                                               |
| -------- | -------------------------------------------------- |
| 설문조사 | 설문지를 통한 시장조사, 고객 만족도 조사 등을 시행 |
| 인터뷰   | 심층적인 정보 또는 개인적 견해를 수집              |
| 관찰조사 | 직접 현장에 나서서 관찰 및 기록                    |

### 레퍼런스(Reference)

작업시 참고할 수 있는 자료를 레퍼런스라고 한다.

프로토타입 제작 과정에서 트랜드 파악을 통해 영감을 얻거나, **산업 표준과 디자인 가이드라인** 을 준수하기 위해 조사한다.

**`이머징 이슈(Emerging issue)`**

드러나기 시작했지만 가시화 되지 않은 문제점

미래에 보편적 또는 대중적 문제점이 될 수 있는 가능성의 형태로, 미래의 시장 동향 및 사용자 경험 해결을 위해 분석된다.

**`레퍼런스 수집 방법`**

| 이름                          | 방법                                         |
| ----------------------------- | -------------------------------------------- |
| 비언어적 요소 조사, 기록 기술 | 학술 논문, 기사 등 문헌 자료를 참고하여 수집 |
| 정보 검색 기술                | 인터넷 검색을 활용하여 수집                  |

### 경쟁사 분석(Competitive analysis)

레퍼런스 수집과 포함관계(subset)에 있으며, 시장조사 단계에 시행된다.

타사 웹사이트의 장단점을 분석하고 차별점을 찾아 자사 웹사이트에 대한 전략을 수립한다.

**`경쟁사 분석 종류`**

| 종류            | 설명                                                                     |
| --------------- | ------------------------------------------------------------------------ |
| 경쟁사 식별     | 주요 경쟁사를 식별하여, 제품이나 서비스의 품질, 가격 등을 분석           |
| 웹사이트 디자인 | 레이아웃 등 분석                                                         |
| 콘텐츠          | 콘텐츠 종류 및 품질, 전달 방식 분석                                      |
| 기능 및 서비스  | 웹사이트 기능 및 서비스의 특징(결제 옵션 및 과정, 고객 지원 서비스) 분석 |
| SEO 분석        | 경쟁사의 SEO 전략 분석                                                   |
| 리뷰 및 피드백  | 경쟁사 고객의 리뷰와 피드백을 검토하여 만족하는 부분과 불만사항을 파악   |

**`경쟁사 분석 방법`**

| 종류                   | 설명                                                                          |
| ---------------------- | ----------------------------------------------------------------------------- |
| SWOT 분석              | 강점(Strengths) / 약점(Weaknesses) / 기회(Opportunities) / 위협(Threats) 분석 |
| 벤치마킹(Benchmarking) | 우수 사이트를 타겟으로 하여 개선 방안 도출                                    |
| 시장 조사              | 경쟁사 사이트의 트래픽, 브랜드 인지도 등 조사                                 |
| 웹사이트 디자인 분석   | 경쟁사 웹사이트의 UX/UI 분석                                                  |

### 웹사이트 사용자 유형

| 종류                             | 설명                                                                                                                           |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| 프로슈머(Prosumer)               | 생산자(Producer)와 소비자(Consumer)의 합성어. 소비자인 동시에 생산자 역할. 웹디자인의 생산과 소비과정에서 능동적인 역할을 한다 |
| 정보 탐색자(Information Seeker)  | 정보를 찾기 위해 방문하는 사용자                                                                                               |
| 충성 고객(Loyal consumer)        | 지속적으로 관심을 가지며 방문하는 사용자                                                                                       |
| 피드백 제공자(Feedback provider) | 리뷰, 댓글 등을 통해 피드백을 제공하는 사용자                                                                                  |
| 인플루언서(Influencer)           | 온라인에서 영향력이 있는 사용자                                                                                                |
| 콘텐츠 제작자(Content creator)   | 블로그, 비디오 등 콘텐츠를 제공하는 사용자                                                                                     |
| 지원 요청자(Support seeker)      | 제품, 서비스와 관련한 지원을 받기 위해 방문하는 사용자                                                                         |
| 구독자(Subscriber)               | 프리미엄 콘텐츠를 받기 위해 구독하는 사용자                                                                                    |

## 프로젝트 개념 및 구조

### 프로젝트

**`프로젝트의 특성`**

| 종류           | 설명                                                                 |
| -------------- | -------------------------------------------------------------------- |
| 일시성(한시성) | 시작과 끝이 명확하여, 일정기간 동안만 지속된다                       |
| 고유성         | 일상적인 운영 활동과는 독립적으로 고유한 결과물을 생산하기 위해 수행 |
| 점진적 상세화  | 세부사항은 프로젝트가 진행되면서 구체화됨                            |

### PMBOK(Project Management Body of Knowledge)

PMBOK는 **PMI(Project Management Institute)** 에서 발행한 프로젝트 관리에 관한 지식과 표준을 체계적으로 정리한 지침서이다.

**`PMBOK의 5단계 프로세스`**

| 단계             | 설명                                 |
| ---------------- | ------------------------------------ |
| 시작             | 프로젝트 승인 및 시작                |
| 계획             | 프로젝트 범위 설정 및 목표 구체화    |
| 실행             | 프로젝트 수행                        |
| 모니터링 및 제어 | 진행 상황 및 성과 추적/변경          |
| 종료             | 최종 결과물 전달 / 문서 작성 및 보관 |

**`PMBOK에 기반한 웹디자인 프로세스`**

기획 &rarr; 웹사이트 기획 &rarr; 사이트 구축 디자인 및 개발 &rarr; 테스트 및 디버깅 &rarr; 배포, 홍보, 유지보수

### 프로젝트 관련 용어

| 용어                  | 설명                                                     |
| --------------------- | -------------------------------------------------------- |
| 성과(Outcome)         | 프로젝트 완료 후 얻어지는 최종 결과                      |
| 아티펙트(Artifact)    | 프로젝트 진행 중 얻어지는 모든 종류의 산출물, 문서, 도구 |
| 포트폴리오(Portfolio) | 목표를 달성하기 위해 그룹으로 관리되는 프로젝트          |

### 프로젝트 관련 문서

| 종류                                    | 설명                                                                                     |
| --------------------------------------- | ---------------------------------------------------------------------------------------- |
| 제안 요청서(RFP - Request for Proposal) | 프로젝트 발주기관이 공급 업체에 제안서 제출을 요청하기 위해 작성                         |
| 제안서                                  | 서비스 공급 업체가 발주기관에 제출하는 문서. 차별화 전략, 계획, 수행능력, 예산 등을 표현 |
| 프로젝트 계획서                         | 프로젝트 제안서 승인 후, 상세 계획을 세우기 위해 작성                                    |
| 최종 보고서                             | 프로젝트 종료시 작성. 프로젝트 전체 과정을 정리하고 성과 평가                            |

### 마르미(MaRMI) 아키텍처

**MaRMI(Magic and Robust Methodology Integrated)** 는 국내에서 개발된 한국형 정보시스템 개발 방법론이다.

여러 단계를 거쳐 발전해 왔으며, 최신 버전인 MaRMI-III는 CBD(Component-based Development)에 중점을 두고 있다.

**`MaRMI 공정 7단계`**

개발계획 &rarr; 요구 분석 &rarr; 기본 설계 &rarr; 상세 설계 &rarr; 구축 &rarr; 테스트 &rarr; 설치 및 인도

**`MaRMI 특징`**

| 특징                                  | 설명                                           |
| ------------------------------------- | ---------------------------------------------- |
| 국제 표준 수용                        | ISO/IEC 12207 개발 공정 기반                   |
| 개발 공정의 계층화                    | 개발 과정을 체계적으로 나누어 관리             |
| UML / EJB(Enterprise Java Beans) 기반 | 현재 EJB는 spring boot / JPA(hibernate)로 대체 |

**`MaRMI III 구성요소`**

| 요소        | 설명                                                                                                        |
| ----------- | ----------------------------------------------------------------------------------------------------------- |
| 개요서      | MaRMI 방법론의 전반적인 개요를 제공하는 문서                                                                |
| 절차서      | 프로젝트와 관련된 필요한 활동과 작업을 정의한 문서. 계획 / 아키텍처 / 점진적 개발 / 인도 총 4개 단계로 구성 |
| 기법서      | 프로젝트 진행시 사용할 수 있는 기법과 도구를 설명한 문서                                                    |
| 양식 정의서 | 프로젝트 진행중 생성되는 산출물의 양식을 정의한 문서                                                        |
| 적용 사례서 | 프로젝트에 적용한 외부 사례를 설명한 문서                                                                   |

### WBS(Work Breakdown Structure)

작업분류 체계(WBS)란, 프로젝트를 구성요소로 분할하여 체계적으로 관리하는 방법 또는 체계를 말한다.

일반적으로 트리구조 다이어그램 형태로 시각적으로 표현된다.

- 최상위 수준에 프로젝트 산출물이 위치하고, 각 단계마다 주요 산출물이 정의된다.

- WBS다이어그램 또는 간트차트 등이 활용된다.

## 산업재산권과 저작권

### 산업재산권

산업재산권은 상업적 또는 산업적 활동과 관련된 지적재산권을 뜻한다.

**`산업재산권 종류`**

| 종류                        | 설명                                                                        | 존속 기간                       |
| --------------------------- | --------------------------------------------------------------------------- | ------------------------------- |
| 특허권(Big invention)       | 고도 발명에 대해 독점적인 권리                                              | 출원일로부터 20년               |
| 실용신안권(Small invention) | 작은 개량, 기술적 소발명에 대한 권리                                        | 출원일로부터 10년               |
| 상표권                      | 특정 상품이나 서비스에 대한 상표, 로고, 브랜드명 등 고유의 표시에 대한 권리 | 최초 등록일부터 10년(갱신 가능) |
| 디자인권                    | 제품의 외관, 형태, 디자인에 대한 독점적인 권리                              | 출원일로부터 20년(2023년 개정)  |

**`특허권 / 실용신안권 차이점`**

특허권은 모든 물리적 / 비물리적 산출물에 대한 기술적 발명 범주에 해당되지만, 실용신안권은 실제로 물리적 **제품** 에 대한 기술에 한정된다.

| 구분      | 특허권                    | 실용신안권           |
| --------- | ------------------------- | -------------------- |
| 보호 대상 | 발명(물건·방법·공정 모두) | 고안(물건·구조만)    |
| 수준      | 고도한 기술적 창작        | 상대적으로 낮아도 됨 |
| 심사      | 까다로운 실질심사         | 무심사 등록 후 심사  |
| 존속기간  | 출원일로부터 20년         | 출원일로부터 10년    |
| 비용      | 상대적으로 높음           | 상대적으로 낮음      |

**`디자인권 범위`**

| 등록 가능한 사례                   | 등록 불가능한 사례                                            |
| ---------------------------------- | ------------------------------------------------------------- |
| 스마트폰의 둥근 모서리 형상과 배치 | 기능에서 비롯된 형상 (그 모양이어야만 기능이 되는 것)         |
| 콜라병의 독특한 곡선 형태          | 물품이 아닌 것 (건물 외관 등 — 단, 건축디자인은 별도 논의 중) |
| 운동화 밑창의 패턴·모양            | 동적인 움직임만 있는 것 (단, 화상디자인은 가능)               |
| 자동차 헤드라이트 디자인           |                                                               |
| 아이콘·UI 화면 배치 (화상디자인)   |                                                               |

### 저작권

저작권은 창작자가 자신의 저작물(창작물)에 대해 가지는 **배타적 권리** 를 말한다.

저작자는 인격적 권리인 **저작인격권**과 재산적 권리인 **저작재산권** 을 가진다.

**`저작인격권`**

저작인격권은 비재산적 권리로, 저작자의 인격을 보호하기 위한 권리다.

| 종류         | 설명                                                           |
| ------------ | -------------------------------------------------------------- |
| 공표권       | 저작물의 최초 공표 여부, 시기를 결정할 권리                    |
| 성명표시권   | 저작물에 이름을 표시할 권리, 익명 또는 가명으로 발표할 수 있음 |
| 동일성유지권 | 저작물이 수정되는 것으로부터 보호하는 권리                     |

**`저작재산권`**

저작재산권이란, 자신의 저작물을 통해 경제적 이익을 얻을 수 있도록 보호하는 권리다.

저작재산권의 존속기간은 **저작자 생존기간 + 사후 70년** 이다. (저작인격권과 달리 양도 가능)

| 종류               | 설명                                                                       |
| ------------------ | -------------------------------------------------------------------------- |
| 복제권             | 복제할 수 있는 권리                                                        |
| 배포권             | 판매 또는 무료로 배포 등 유통할 수 있는 권리                               |
| 공연권             | 저작물을 상연, 연주, 가창, 상영 등의 방법으로 공중에게 공개할 수 있는 권리 |
| 공중송신권         | 대중이 수신하거나 접근할 수 있게 하는 권리                                 |
| 전시권             | 미술 작품을 공개적으로 전시할 수 있는 권리                                 |
| 대여권             | 영리를 목적으로 타인에게 대여할 수 있는 권리                               |
| 2차적저작물 작성권 | 기존 저작물을 바탕으로 수정을 통해 새로운 저작물을 제작할 수 있는 권리     |

**`지적재산권의 제한`**

특정 상황에서는 지적재산권이 제한된다.

| 종류                     | 설명                                            |
| ------------------------ | ----------------------------------------------- |
| 사적 이용을 통한 복제    | 개인적인 목적으로 복제하는 경우                 |
| 보도, 비평, 교육 등 이용 | 공정 이용으로 간주될 경우                       |
| 도서관 등에서의 복제     | 공공적인 목적으로 운영되는 기관에서 이용될 경우 |
| 장애인을 위한 복제       | 장애인을 위해 필수적인 목적으로 이용될 경우     |

**`저작인접권`**

저작인접자란, 저작자 외에 저작물과 관련된 활동을 통해 권리를 가지는 사람을 말한다.

저작인접자는 **저작인접권** 을 가진다.

| 종류              | 설명                                                                                                                             |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| 실연자의 권리     | 음악, 공연과 같은 실연이 가능한 저작물에 대해, 저작물을 실연하는 실연자가 자신의 실연을 보호받을 수 있는 권리                    |
| 음반제작자의 권리 | 음반 제작에 기여한 기업, 사람들이 그 음반에 대한 복제, 배포, 대여 등에 대해 보호받을 권리                                        |
| 방송사업자의 권리 | 방송 프로그램 제작에 기여한 제작자 또는 사업자가 그 방송 프로그램에 대해 보호받을 권리. 재송신, 녹화, 복제 등에 대한 권리가 포함 |

**`공공저작물`**

정부, 공공기관, 또는 기타 공익을 위해 만들어진 저작물.

**공공누리(KOGL)** 제도에 따라 이용 조건이 부여되며, 출처 표시 등 조건을 지켜야 한다.

단, 법령·고시·판결문 등은 저작권법 제7조에 의해 저작권 보호 대상에서 제외되어 자유롭게 이용 가능하다.
