## 운영체제의 개요

### 운영체제의 정의

운영체제는 컴퓨터 사용자와 컴퓨터 하드웨어 간의 Interface로써 동작하는 시스템 SW다.

OS는 **스스로 기능을 수행하지 않고** , 다른 응용 프로그램이 작업을 할 수 있도록 환경을 마련한다.

### OS 성능 평가 항목

**`처리량(Throughput)`**

주어진 시간 내에 처리되는 작업의 양.

Throughput이 높을수록 OS 성능이 좋다.

**`응답 시간(Response Time)`**

사용자가 시스템에 명령을 입력한 뒤 그 결과가 출력될 때까지 걸리는 시간.

Response Time이 짧을수록 OS 성능이 좋다.

**`신뢰성(Reliability)`**

주어진 작업에 대해서 얼마나 오류 없이 처리하는 지에 대한 척도.

**`사용 가능도(Availability)`**

시스템 운영 시간 중 얼마나 많은 시간을 사용 가능한지에 대한 척도.

### OS 기능

메모리 상태관리, 사용자간의 자원공유 등의 기능을 제공.

### OS 운영 방식

**`일괄처리 시스템(Batch Processing System)`**

유사한 성격의 작업을 한 번에 모아 처리하는 시스템.

오프라인 시스템에서 사용

**`다중 프로그래밍 시스템(Multi Programming System)`**

CPU처리 효율을 극대화 하기 위해 하나 이상의 프로그램을 RAM에 적재하고 처리하는 방식

**`시분할 시스템(Time Sharing System)`**

**하나의 컴퓨터를 여러개의 단말기** 가 공동으로 사용하는 시스템.

단말기 사용자를 위한 대화형 처리를 위해 개발되었으며, **실시간 응답** 이 요구된다.

**`다중 처리 시스템(Multi Processing System)`**

**여러 개의 CPU** 와 한 개의 주기억장치로 여러 프로그램을 동시에 처리하는 시스템.

**`실시간 처리 시스템(Real Time Processing System)`**

작업이 발생한 시점에서 즉각적으로 처리하는 시스템.

**`다중 모드 시스템(Multi Mode System)`**

일괄처리, 시분할, 다중처리, 실시간 처리를 함께 사용하는 시스템

**`분산 처리 시스템(Distributed Processing System)`**

여러 대의 컴퓨터들에 의해 작업을 나누어 처리하고, 결과를 통신망을 이용해 공유하는 시스템.

### OS 구성

OS는 제어 프로그램(Control Program)과 처리 프로그램(Processing Program)으로 구분할 수 있다.

제어 프로그램은 감시, 작업제어, 데이터 관리 프로그램으로 구성된다.

처리 프로그램은 언어 번역, 서비스, 문제 처리 프로그램으로 구성된다.

### 제어 프로그램(Control Program)

**`감시 프로그램(Supervisor Program)`**

자원의 할당 및 시스템 전체의 작동상태를 감시, 감독하는 프로그램.

- 인터럽트 처리기 — 인터럽트 발생 시 원인 파악 후 적절한 처리 루틴으로 분기

- CPU 스케줄러 — 어떤 프로세스에 CPU를 줄지 감시·결정

- 메모리 감시 모듈 — 메모리 사용 현황을 감시하고 오류 감지

- 시스템 콜 핸들러 — 사용자 프로그램의 OS 요청을 감시하고 처리

**`작업 제어 프로그램(Job Control Program)`**

스케줄링 및 시스템 자원 할당 기능을 수행.

- 프로세스 스케줄러 — 준비 큐의 프로세스에 CPU 할당 순서 결정 (FCFS, Round Robin 등)

- JCL(Job Control Language) 처리기 — 대형 메인프레임에서 작업 순서와 자원 요구를 기술한 명령
  처리
- 배치 작업 관리자 — 여러 배치 작업의 실행 순서와 우선순위 관리

- 자원 할당 모듈 — 프린터, 디스크 등 자원을 각 작업에 배분

**`데이터 관리 프로그램(Data Management Program)`**

주기억장치와 보조기억장치 간의 자료 전송, 파일의 조작 및 처리, 프로그램 간의 논리적 연결 등 시스템에서 취급하는 파일과 데이터를 표준적인 방법으로 처리할 수 있도록 관리.

- 파일 시스템(FAT, NTFS, ext4) — 파일 생성·삭제·읽기·쓰기를 표준 방법으로 관리

- 버퍼 관리자 — 주기억장치↔보조기억장치 간 데이터 전송 시 버퍼를 통해 속도 차이 완충

- 입출력 관리자(I/O Manager) — 디스크, 테이프 등 장치와의 데이터 입출력 처리

- DBMS(데이터베이스 관리 시스템) — 데이터를 구조화하여 저장·조회·수정 관리

### 처리 프로그램(Processing Program)

**`언어 번역 프로그램(Language Translator Program)`**

원시 프로그램을 번역하여 목적 프로그램을 생성.

Assembler, Compiler가 이에 해당.

**`서비스 프로그램(Service Program)`**

시스템 제공자가 사용 빈도가 높은 프로그램을 미리 작성하여 사용자에게 제공.

시스템 제공자가 미리 만들어 제공하는 유틸리티성 프로그램.

- 정렬/병합 프로그램 — 데이터를 오름차순/내림차순으로 정렬

- 링커(Linker) — 목적 파일들을 연결해 실행 파일 생성

- 로더(Loader) — 프로그램을 메모리에 올려 실행 준비

- 디버거(Debugger) — 프로그램 오류 탐지 및 수정 지원

- 텍스트 에디터 — 코드/문서 편집 (예: vi, notepad)

- 파일 압축 프로그램 — 파일 압축/해제 유틸리티

**`문제 프로그램(Problem Program)`**

특정 업무 처리를 위해 사용자 또는 기업이 직접 작성한 프로그램.

- 급여 관리 프로그램 — 직원 급여 계산 및 명세서 출력

- 재고 관리 프로그램 — 물품 입출고 및 재고 현황 관리

- 성적 처리 프로그램 — 학생 성적 입력, 평균 계산, 출력

- 회계 프로그램 — 기업 매출/지출 내역 관리

- 예약 시스템 — 항공권, 호텔 등 예약 처리
